10.1¿Qué es el SEO?

SEO (Search Engine Optimization) es el conjunto de técnicas que mejoran la visibilidad de una web en los resultados orgánicos de los buscadores. Desde el HTML podemos influir en cómo Google y otros motores interpretan y muestran nuestro contenido.

Los 3 pilares del SEO

  • SEO Técnico: estructura del HTML, velocidad de carga, indexabilidad
  • SEO On-page: contenido, palabras clave, metadatos, estructura semántica
  • SEO Off-page: enlaces externos, autoridad del dominio (no se controla desde HTML)

En este tema nos centramos en los dos primeros pilares: lo que podemos controlar directamente desde el código HTML.

10.2La etiqueta <title>

Es el factor on-page más importante. Aparece en la pestaña del navegador y como título azul en los resultados de búsqueda.

Buenas prácticas

  • Único por cada página
  • 50-60 caracteres máximo
  • Palabra clave principal al inicio
  • Formato: Palabra clave - Marca
  • Descriptivo y atractivo para el clic

Errores comunes

  • Título genérico: «Inicio» o «Página 1»
  • Repetir el mismo título en todas las páginas
  • Exceder los 60 caracteres (se corta en Google)
  • Rellenar con palabras clave (keyword stuffing)
<!-- Mal -->
<title>Inicio</title>
<title>Venta de zapatos zapatos baratos comprar zapatos</title>

<!-- Bien -->
<title>Zapatos de Running para Mujer - TiendaDeporte</title>
<title>Guía HTML: Estructura Básica - Apuntes del Curso</title>

10.3Meta description

Aparece como texto descriptivo bajo el título en los resultados de búsqueda. No afecta directamente al ranking pero influye enormemente en el CTR (Click Through Rate).

  • Máximo 155-160 caracteres
  • Única por cada página
  • Incluir la palabra clave de forma natural
  • Incluir una llamada a la acción (CTA)
  • Describir fielmente el contenido de la página
<meta name="description"
      content="Aprende la estructura básica de HTML5 con ejemplos
               prácticos. Guía completa para principiantes
               con ejercicios interactivos.">

Simulación en Google

https://ejemplo.com/apuntes/html/estructura

Aprende la estructura básica de HTML5 con ejemplos prácticos. Guía completa para principiantes con ejercicios interactivos.

10.4Estructura de encabezados para SEO

Los buscadores usan los encabezados para entender la estructura temática de la página. Una jerarquía bien definida mejora la indexación y puede generar rich snippets.

Mal: jerarquía rota

<h1>Mi blog</h1>
<h3>Artículo 1</h3>   <!-- saltó h2 -->
<h1>Otro título</h1>  <!-- dos h1 -->
<h4>Sección</h4>       <!-- saltó h2 y h3 -->

Bien: jerarquía lógica

<h1>Recetas de Pasta Italiana</h1>
  <h2>Carbonara Clásica</h2>
    <h3>Ingredientes</h3>
    <h3>Preparación</h3>
  <h2>Pesto Genovese</h2>
    <h3>Ingredientes</h3>
    <h3>Preparación</h3>

Reglas clave

  • Un solo <h1> por página, con la palabra clave principal
  • No saltar niveles: h1 → h2 → h3 (no h1 → h3)
  • Los <h2> definen las secciones principales
  • Usar palabras clave secundarias en los <h2> y <h3>
  • Que los encabezados sean descriptivos, no genéricos

10.5Meta tags esenciales para SEO

El <head> contiene metadatos críticos para los buscadores y redes sociales.

10.5.1 Meta robots

Controla cómo los buscadores indexan y rastrean la página.

Directiva Efecto
indexPermite indexar (por defecto)
noindexNo indexar esta página
followSeguir los enlaces de la página (por defecto)
nofollowNo seguir los enlaces
nosnippetNo mostrar fragmento en resultados
 
<!-- Página normal: indexable y rastreable -->
<meta name="robots" content="index, follow">

<!-- Página privada: no indexar -->
<meta name="robots" content="noindex, nofollow">

10.5.2 URL canónica

Evita contenido duplicado indicando cuál es la URL «oficial» de una página.

<!-- Esta página puede tener parámetros: ?color=rojo&talla=42 -->
<!-- Canonical apunta a la URL limpia -->
<link rel="canonical" href="https://tienda.com/zapatos/modelo-x">

10.5.3 Open Graph (redes sociales)

Controla cómo se muestra el enlace cuando se comparte en redes sociales.

<!-- Open Graph (Facebook, LinkedIn, WhatsApp) -->
<meta property="og:title" content="Guía HTML: Estructura Básica">
<meta property="og:description" content="Aprende HTML5 desde cero">
<meta property="og:image" content="https://ejemplo.com/img/portada.jpg">
<meta property="og:url" content="https://ejemplo.com/apuntes/html">
<meta property="og:type" content="article">

<!-- Twitter Card -->
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="Guía HTML: Estructura Básica">
<meta name="twitter:description" content="Aprende HTML5 desde cero">
<meta name="twitter:image" content="https://ejemplo.com/img/portada.jpg">

10.6Enlaces optimizados para SEO

Los enlaces transmiten autoridad (link juice) y ayudan a los buscadores a descubrir y clasificar contenido.

Atributo Uso Ejemplo
rel="nofollow" No transmitir autoridad Enlaces de publicidad, comentarios de usuarios
rel="sponsored" Enlace patrocinado/pagado Banners publicitarios, afiliados
rel="ugc" Contenido generado por usuarios Foros, comentarios, reseñas
rel="noopener" Seguridad con target="_blank" Siempre con enlaces externos nuevos

Mal: texto de enlace

<a href="/guia">Clic aquí</a>
<a href="/guia">Leer más</a>
<a href="/guia">Enlace</a>

Bien: texto descriptivo

<a href="/guia">Guía completa de HTML5</a>
<a href="/guia">Leer la guía de HTML</a>
<a href="https://ads.com"
   rel="sponsored noopener"
   target="_blank">Oferta especial</a>

10.7Imágenes optimizadas para SEO

Las imágenes son una fuente importante de tráfico a través de Google Images. Optimizarlas mejora el SEO y la velocidad de carga.

  • Nombre descriptivo: zapatos-running-mujer.webp en vez de IMG_4521.jpg
  • Alt descriptivo: incluir palabra clave de forma natural
  • Formato moderno: WebP o AVIF para mejor compresión
  • Dimensiones correctas: definir width y height para evitar CLS
  • Carga diferida: usar loading="lazy" para imágenes fuera del viewport
<!-- Imagen optimizada para SEO -->
<img src="zapatos-running-mujer.webp"
     alt="Zapatos de running para mujer Nike Air Zoom en color rosa"
     width="800"
     height="600"
     loading="lazy">

<!-- Imagen responsive con múltiples tamaños -->
<picture>
  <source srcset="zapatos-800.avif" type="image/avif">
  <source srcset="zapatos-800.webp" type="image/webp">
  <img src="zapatos-800.jpg"
       alt="Zapatos de running para mujer"
       width="800" height="600"
       loading="lazy">
</picture>

10.8Datos estructurados (Schema.org)

Los datos estructurados ayudan a Google a entender el significado del contenido y a generar rich snippets (resultados enriquecidos): estrellas de valoración, precios, recetas, FAQs, etc.

Ejemplo: artículo con JSON-LD

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Guía de HTML para principiantes",
  "author": {
    "@type": "Person",
    "name": "Nombre del autor"
  },
  "datePublished": "2025-01-15",
  "description": "Aprende los fundamentos de HTML5...",
  "image": "https://ejemplo.com/img/portada.jpg"
}
</script>

Ejemplo: FAQ (preguntas frecuentes)

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "¿Qué es HTML?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "HTML es el lenguaje estándar para crear páginas web."
      }
    }
  ]
}
</script>

Herramienta de validación

Usa Rich Results Test de Google para verificar que tus datos estructurados son correctos y generarán rich snippets.

10.9Rendimiento HTML y Core Web Vitals

Google usa las Core Web Vitals como factor de ranking. Varias de estas métricas se pueden mejorar directamente desde el HTML.

Métrica Qué mide Optimización HTML
LCP Tiempo del elemento más grande visible <link rel="preload"> para la imagen principal
CLS Desplazamientos inesperados del layout Definir width/height en imágenes e iframes
INP Retraso de interacción <script defer> para no bloquear el render 
<head>
  <!-- Precargar la imagen hero (mejora LCP) -->
  <link rel="preload" as="image" href="hero.webp">

  <!-- Preconectar a orígenes externos -->
  <link rel="preconnect" href="https://fonts.googleapis.com">

  <!-- CSS crítico inline, resto diferido -->
  <style>/* CSS crítico */</style>
  <link rel="stylesheet" href="estilos.css" media="print"
        onload="this.media='all'">

  <!-- Scripts diferidos (no bloquean render) -->
  <script src="app.js" defer></script>
</head>

10.10Checklist SEO para HTML

Head y metadatos

  • □ <title> único y descriptivo (50-60 chars)
  • □ meta description única (155 chars)
  • □ meta viewport configurado
  • □ URL canónica definida
  • □ Open Graph y Twitter Cards
  • □ Favicon configurado
  • □ lang definido en <html>

Contenido y estructura

  • □ Un solo <h1> con palabra clave
  • □ Jerarquía de encabezados correcta
  • □ Imágenes con alt, width, height
  • □ Enlaces con texto descriptivo
  • □ Datos estructurados (JSON-LD)
  • □ Estructura semántica (header, main, nav, footer)
  • □ Scripts con defer o async

Herramientas de auditoría SEO

  • Google Lighthouse: auditoría integrada en Chrome DevTools (F12 → Lighthouse)
  • Google Search Console: monitoriza el rendimiento en resultados de búsqueda
  • Rich Results Test: valida datos estructurados
  • PageSpeed Insights: analiza Core Web Vitals en móvil y escritorio
