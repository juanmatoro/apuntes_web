8.1¿Qué son los atributos globales?

Los atributos globales son atributos que pueden aplicarse a cualquier elemento HTML, independientemente de su tipo. A diferencia de los atributos específicos (como src en <img> o href en <a>), los globales son universales.

Atributos globales más utilizados

  • id — identificador único
  • class — clase(s) CSS
  • style — estilos en línea
  • title — tooltip informativo
  • data-* — datos personalizados
  • hidden — ocultar elemento
  • lang — idioma del contenido
  • dir — dirección del texto
  • tabindex — orden de tabulación
  • contenteditable — contenido editable
  • draggable — elemento arrastrable
  • spellcheck — corrector ortográfico
  • translate — traducibilidad

Sintaxis general

<elemento atributo="valor">Contenido</elemento>

<!-- Atributos booleanos: solo con estar presentes se activan -->
<input type="text" required disabled>

<!-- Equivale a: -->
<input type="text" required="required" disabled="disabled">

8.2id y class

Son los dos atributos más usados en HTML. Permiten identificar y clasificar elementos para aplicarles estilos CSS o manipularlos con JavaScript.

id — Identificador único

  • Debe ser único en todo el documento
  • No puede contener espacios
  • Debe empezar con una letra (a-z, A-Z)
  • Se usa como ancla: href="#id"
  • Se selecciona con #id en CSS
  • En JS: getElementById("id")

class — Clasificación

  • Puede repetirse en múltiples elementos
  • Un elemento puede tener varias clases separadas por espacio
  • Se selecciona con .clase en CSS
  • En JS: getElementsByClassName()
  • Es la forma principal de aplicar estilos reutilizables
  • Convención: usar kebab-case (ej: mi-clase)

Ejemplos de uso

<!-- id: identificador único para ancla y JS -->
<section id="contacto">
  <h2>Contáctanos</h2>
</section>
<a href="#contacto">Ir a contacto</a>

<!-- class: una o varias clases -->
<div class="tarjeta destacada">
  <p class="tarjeta-titulo">Título</p>
  <p class="tarjeta-texto">Contenido</p>
</div>

<!-- Combinar id y class -->
<nav id="menu-principal" class="nav-horizontal activo">
  ...
</nav>

Mal

<!-- id duplicado -->
<div id="caja">A</div>
<div id="caja">B</div>

<!-- id con espacios -->
<div id="mi caja"></div>

<!-- id empieza con número -->
<div id="1-seccion"></div>

Bien

<!-- id únicos -->
<div id="caja-a">A</div>
<div id="caja-b">B</div>

<!-- id con guión -->
<div id="mi-caja"></div>

<!-- id empieza con letra -->
<div id="seccion-1"></div>

8.3style — CSS en línea

El atributo style permite aplicar estilos CSS directamente sobre un elemento. Tiene la mayor especificidad en CSS (supera a selectores de clase e id).

Especificidad CSS (de menor a mayor)

Selector Especificidad Ejemplo
Elemento 0-0-1 p { }
Clase 0-1-0 .texto { }
ID 1-0-0 #titulo { }
Inline (style="") 1-0-0-0 style="color:red" 
<!-- CSS inline: se aplica solo a este elemento -->
<p style="color: red; font-size: 18px; margin-bottom: 10px;">
  Texto con estilos en línea
</p>

<!-- Múltiples propiedades separadas por punto y coma -->
<div style="background: #f5f5f5; padding: 20px; border-radius: 8px;">
  Caja con estilos
</div>

Evitar

<!-- Repetir estilos inline -->
<p style="color:blue;font-size:14px">A</p>
<p style="color:blue;font-size:14px">B</p>
<p style="color:blue;font-size:14px">C</p>

Duplica código y es difícil de mantener

Preferir

<!-- Usar clases reutilizables -->
<p class="texto-azul">A</p>
<p class="texto-azul">B</p>
<p class="texto-azul">C</p>

/* En CSS */
.texto-azul { color:blue; font-size:14px }

Un solo lugar para cambiar estilos

8.4title — Tooltip informativo

El atributo title muestra un tooltip (cuadro de texto emergente) al pasar el cursor sobre un elemento. Proporciona información adicional sin ocupar espacio en la página.

  • Se muestra al hacer hover con el ratón
  • En <abbr> se usa para mostrar el significado completo de una abreviatura
  • En <a> aporta contexto extra sobre el enlace
  • No sustituye al atributo alt en imágenes ni a aria-label para accesibilidad
  • No es accesible por teclado ni en dispositivos táctiles

Ejemplo interactivo

La W3C define los estándares web.

Enlace con tooltip — pasa el ratón para ver

Pasa el mouse por encima de este texto

<!-- En abreviaturas -->
<abbr title="World Wide Web Consortium">W3C</abbr>

<!-- En enlaces -->
<a href="/contacto" title="Ir al formulario de contacto">Contacto</a>

<!-- En cualquier elemento -->
<p title="Información adicional">Texto visible</p>

8.5data-* — Atributos personalizados

Los atributos data-* permiten almacenar datos personalizados directamente en los elementos HTML. Son invisibles para el usuario pero accesibles desde CSS y JavaScript.

Reglas de nomenclatura

  • Siempre empiezan con data-
  • Solo minúsculas, números y guiones
  • Mínimo un carácter después de data-
  • Sin mayúsculas (se convierte a camelCase en JS)
  • Ejemplo: data-user-id → dataset.userId en JS

En HTML

<!-- Almacenar información en elementos -->
<article
  data-id="42"
  data-category="tecnologia"
  data-author-name="Ana"
  data-published="true">
  <h2>Título del artículo</h2>
</article>

<button data-action="delete" data-target-id="42">
  Eliminar
</button>

Acceso desde JavaScript

const art = document.querySelector('article');

// Leer datos con dataset
art.dataset.id;          // "42"
art.dataset.category;    // "tecnologia"
art.dataset.authorName;  // "Ana"

// Modificar datos
art.dataset.category = "ciencia";

// Alternativa: getAttribute
art.getAttribute('data-id'); // "42"

Acceso desde CSS

/* Seleccionar por atributo data-* */
[data-category="tecnologia"] {
  border-left: 3px solid blue;
}

/* Mostrar valor en contenido */
[data-status]::after {
  content: attr(data-status);
}

/* Atributo presente */
[data-published] {
  opacity: 1;
}

/* Atributo NO presente */
:not([data-published]) {
  opacity: 0.5;
}

Casos de uso comunes

  • Configuración de componentes: data-modal="login", data-carousel-speed="3000"
  • Identificación: data-user-id="123", data-product-sku="XYZ"
  • Estado: data-state="open", data-loading="true"
  • Analítica: data-track-event="click-cta"
  • Testing: data-testid="submit-button"

8.6Internacionalización: lang, dir, translate

Estos atributos controlan cómo el navegador, los lectores de pantalla y los traductores interpretan el idioma y la dirección del texto.

lang

Define el idioma del contenido. Usa códigos ISO 639-1.

  • es — Español
  • en — Inglés
  • fr — Francés
  • ar — Árabe
  • ca — Catalán

dir

Dirección del texto.

  • ltr — Izquierda a derecha (por defecto)
  • rtl — Derecha a izquierda (árabe, hebreo)
  • auto — El navegador decide

translate

Indica si el contenido debe traducirse.

  • yes — Traducir (defecto)
  • no — No traducir (marcas, código, nombres propios)
<!-- Idioma del documento completo -->
<html lang="es">

<!-- Texto en otro idioma dentro del documento -->
<p>En inglés se dice <span lang="en">Hello World</span></p>

<!-- Texto en árabe (derecha a izquierda) -->
<p lang="ar" dir="rtl">مرحبا</p>

<!-- No traducir nombres de marca -->
<p>Usamos <span translate="no">Google Chrome</span> para navegar.</p>

<!-- No traducir código -->
<code translate="no">console.log("hello")</code>

8.7Atributos de interacción

HTML incluye atributos globales que modifican la interactividad de los elementos sin necesidad de JavaScript.

hidden — Ocultar elemento

Oculta el elemento del renderizado. El contenido sigue en el DOM pero no es visible ni ocupa espacio.

<!-- Elemento oculto (booleano) -->
<div hidden>Esto no se ve</div>

<!-- HTML 5.2+: hidden="until-found" (buscable) -->
<div hidden="until-found">
  Se muestra si el usuario busca este texto (Ctrl+F)
</div>

<!-- Mostrar/ocultar con JavaScript -->
<script>
  elemento.hidden = false; // mostrar
  elemento.hidden = true;  // ocultar
</script>

tabindex — Orden de tabulación

Controla si un elemento puede recibir foco con la tecla Tab y en qué orden.

Valor Comportamiento
tabindex="0" Focusable en orden natural del DOM
tabindex="-1" Focusable solo por JavaScript, no con Tab
tabindex="1+" Orden específico (evitar, rompe el flujo natural) 
<!-- Div que puede recibir foco -->
<div tabindex="0">Pulsa Tab para llegar aquí</div>

<!-- Solo accesible por JS (para modales, tooltips...) -->
<div tabindex="-1" id="modal">Modal</div>

contenteditable — Contenido editable

Permite al usuario editar el contenido directamente en el navegador.

Haz clic aquí y edita este texto directamente en el navegador. 
<div contenteditable="true">
  Texto que el usuario puede modificar
</div>

<!-- Hereda del padre -->
<div contenteditable="true">
  <p>Editable</p>
  <p contenteditable="false">NO editable</p>
</div>

draggable

Indica si el elemento puede arrastrarse con el ratón (drag & drop).

<div draggable="true">
  Arrástrame
</div>

<!-- Imágenes y enlaces son
     draggable por defecto -->

spellcheck

Activa o desactiva la revisión ortográfica del navegador.

<!-- Activar en un div editable -->
<div contenteditable="true"
     spellcheck="true">
  Escrbe mal y verás el subrayado
</div>

<!-- Desactivar en campo de código -->
<textarea spellcheck="false">
</textarea>

8.8Atributos de evento (on*)

Los atributos on* permiten ejecutar JavaScript cuando ocurre un evento en el elemento. Aunque funcionan, la buena práctica es usar addEventListener en JavaScript externo.

Eventos de ratón

  • onclick — al hacer clic
  • ondblclick — doble clic
  • onmouseover — cursor entra
  • onmouseout — cursor sale

Eventos de teclado y formulario

  • onkeydown / onkeyup — tecla
  • onfocus / onblur — foco
  • onchange — valor cambia
  • onsubmit — envío de form

Evitar — JS inline

<!-- Mezcla HTML y JS -->
<button onclick="alert('Hola')">
  Clic
</button>

<input onchange="validar(this)">

<!-- Problemas:
  - Difícil de depurar
  - No se puede reutilizar
  - Riesgo de XSS
  - Viola separación de responsabilidades
-->

Preferir — addEventListener

<button id="btn-saludo">
  Clic
</button>

<script>
document
  .getElementById('btn-saludo')
  .addEventListener('click', () => {
    alert('Hola');
  });
</script>

<!-- Ventajas:
  - HTML limpio
  - Fácil de mantener
  - Múltiples listeners
  - Más seguro
-->

8.9Buenas prácticas para escribir HTML

Un HTML bien escrito es más fácil de mantener, más accesible, mejor para SEO y más rápido de cargar. Estas son las convenciones fundamentales.

Estructura del documento

Mal

<html>
<title>Mi página</title>
<h1>Hola</h1>
<p>Texto</p>

<!-- Falta DOCTYPE, head, body,
     charset, viewport, lang -->

Bien

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" ...>
  <title>Mi página</title>
</head>
<body>
  <h1>Hola</h1>
  <p>Texto</p>
</body>
</html>

Semántica sobre presentación

Mal — div para todo

<div class="header">
  <div class="nav">
    <div class="link">Inicio</div>
  </div>
</div>
<div class="main">
  <div class="article">...</div>
</div>
<div class="footer">...</div>

Bien — elementos semánticos

<header>
  <nav>
    <a>Inicio</a>
  </nav>
</header>
<main>
  <article>...</article>
</main>
<footer>...</footer>

Accesibilidad básica

<!-- Siempre alt en imágenes -->
<img src="logo.png" alt="Logo de la empresa">

<!-- Siempre label asociado a inputs -->
<label for="email">Email:</label>
<input type="email" id="email">

<!-- Jerarquía correcta de encabezados -->
<h1>Título principal</h1>       <!-- Solo uno por página -->
  <h2>Sección</h2>
    <h3>Subsección</h3>
  <h2>Otra sección</h2>           <!-- No saltar de h1 a h3 -->

<!-- Idioma del documento -->
<html lang="es">

Separación de responsabilidades

Capa Tecnología Responsabilidad
Estructura HTML Qué es cada cosa (contenido y significado)
Presentación CSS Cómo se ve (colores, layout, tipografía)
Comportamiento JavaScript Qué hace (interactividad, lógica)

Formato y convenciones de código

  • Indentación consistente: usar 2 o 4 espacios (no mezclar con tabs)
  • Minúsculas: etiquetas y atributos siempre en minúsculas (<div>, no <DIV>)
  • Comillas dobles en atributos: class="mi-clase"
  • Cerrar todas las etiquetas: incluso las opcionales como </li>, </p>
  • Autocerrables sin barra: <img>, <br>, <input> (en HTML5 no necesitan />)
  • Comentarios útiles: para secciones grandes, no para lo obvio
  • Un atributo por línea si el elemento tiene muchos atributos
<!-- Comentario útil: marca inicio de sección -->
<!-- === HERO SECTION === -->
<section class="hero">
  <h1>Bienvenido</h1>
</section>

<!-- Elemento con muchos atributos: uno por línea -->
<input
  type="email"
  id="email"
  name="email"
  class="form-input"
  placeholder="tu@email.com"
  required
  autocomplete="email"
>

<!-- Evitar comentarios obvios: -->
<!-- Este es un párrafo -->
<p>Texto</p>

Validación del HTML

Siempre valida tu HTML con herramientas automáticas para detectar errores de sintaxis, etiquetas mal cerradas o atributos inválidos.

Herramientas de validación

  • W3C Validator: validator.w3.org — el validador oficial
  • HTMLHint: extensión para VS Code
  • Lighthouse: auditoría integrada en Chrome DevTools (F12 → Lighthouse)
  • axe DevTools: extensión de navegador para accesibilidad

8.10Checklist de buenas prácticas

Repasa esta lista antes de dar por terminado cualquier documento HTML:

Estructura

  • <!DOCTYPE html> declarado
  • lang en <html>
  • charset y viewport en <head>
  • <title> descriptivo
  • Elementos semánticos usados correctamente
  • Jerarquía de encabezados correcta (h1-h6)

Atributos

  • alt en todas las imágenes
  • <label> asociado a cada input
  • IDs únicos en todo el documento
  • Clases con nombres descriptivos (kebab-case)
  • Sin CSS inline (salvo excepciones justificadas)
  • Sin JS inline (usar addEventListener)

Formato

  • Indentación consistente
  • Etiquetas y atributos en minúsculas
  • Comillas dobles en atributos
  • Todas las etiquetas cerradas
  • Comentarios solo donde aportan valor

Calidad

  • Validado con W3C Validator
  • Sin errores en consola del navegador
  • Separación HTML / CSS / JS
  • Probado en al menos 2 navegadores
  • Responsive (viewport + media queries)
