6.1El elemento <img>

El elemento <img> inserta una imagen en el documento. Es un elemento vacío (no tiene etiqueta de cierre) y requiere como mínimo los atributos src y alt.

Atributos principales

  • src — ruta de la imagen (obligatorio)
  • alt — texto alternativo para accesibilidad y SEO (obligatorio)
  • width / height — dimensiones en píxeles (evita layout shift)
  • loading — lazy (carga diferida) o eager (inmediata, por defecto)
  • decoding — async, sync o auto
  • fetchpriority — high, low o auto (prioridad de descarga)
<!-- Uso básico -->
<img
  src="foto.jpg"
  alt="Descripción de la imagen"
  width="400"
  height="300"
>

<!-- Con carga diferida y prioridad -->
<img
  src="hero.webp"
  alt="Banner principal"
  width="1200"
  height="600"
  loading="lazy"
  decoding="async"
  fetchpriority="high"
>

Mal

<!-- Sin alt: inaccesible -->
<img src="foto.jpg">

<!-- Alt genérico: no aporta -->
<img src="equipo.jpg" alt="imagen">

<!-- Sin dimensiones: causa layout shift -->
<img src="banner.jpg" alt="Banner">

Bien

<!-- Alt descriptivo -->
<img src="foto.jpg"
  alt="Equipo de desarrollo en la oficina">

<!-- Decorativa: alt vacío -->
<img src="adorno.svg" alt="">

<!-- Con dimensiones -->
<img src="banner.jpg" alt="Banner"
  width="1200" height="400">

6.2Formatos de imagen

Elegir el formato correcto es clave para el rendimiento, la calidad visual y la compatibilidad. Cada formato tiene características que lo hacen ideal para situaciones concretas.

Formato Tipo Transparencia Animación Peso Uso ideal
JPEG / JPG Rasterizado No No Medio Fotografías, fondos con degradados
PNG Rasterizado Sí (alfa) No Alto Logos, capturas, imágenes con transparencia
GIF Rasterizado Parcial (1 bit) Sí Variable Animaciones simples, iconos pequeños
SVG Vectorial Sí Sí (CSS/JS) Muy bajo Iconos, logos, gráficos, ilustraciones
WebP Rasterizado Sí Sí Bajo Sustituye JPG/PNG con menos peso (25-35%)
AVIF Rasterizado Sí Sí Muy bajo Máxima compresión, soporte creciente

JPEG / JPG

  • Compresión con pérdida (lossy): reduce peso sacrificando detalle
  • Ideal para fotografías y fondos con muchos colores
  • No soporta transparencia
  • Se puede ajustar la calidad (60-80% es buen equilibrio)
  • Extensiones: .jpg, .jpeg

PNG

  • Compresión sin pérdida (lossless): calidad perfecta
  • Soporta transparencia alfa (256 niveles de opacidad)
  • Ideal para logos, iconos, capturas de pantalla, texto sobre imagen
  • Archivos más pesados que JPG para fotos
  • PNG-8 (256 colores) vs PNG-24 (millones de colores)

SVG

  • Formato vectorial basado en XML
  • Escalable sin perder calidad a cualquier tamaño
  • Se puede editar con CSS y manipular con JavaScript
  • Peso muy reducido para gráficos simples
  • Ideal para iconos, logos, ilustraciones, gráficos
  • Se puede insertar inline en HTML o como <img>

WebP

  • Creado por Google como sucesor de JPG y PNG
  • Soporta compresión con y sin pérdida
  • Soporta transparencia y animación
  • 25-35% más ligero que JPG a calidad equivalente
  • Soporte en todos los navegadores modernos (desde 2020)

GIF

  • Limitado a 256 colores
  • Soporta animación (fotogramas)
  • Transparencia binaria (sí o no, sin niveles)
  • Peso elevado para animaciones largas
  • Mejor alternativa para animaciones: WebP o <video>

AVIF

  • Formato de última generación (basado en AV1)
  • Compresión superior a WebP y JPG
  • Soporta transparencia, HDR y animación
  • Soporte creciente (Chrome, Firefox, Safari 16.4+)
  • Usar con <picture> como formato preferente con fallback

Guía rápida: ¿qué formato uso?

  • Fotografía → WebP (con fallback JPG)
  • Logo / icono → SVG
  • Imagen con transparencia → WebP o PNG
  • Captura de pantalla → PNG
  • Animación corta → WebP animado o GIF
  • Máxima compresión → AVIF (con fallback WebP/JPG)
  • Gráfico editable con CSS/JS → SVG inline

6.3Rasterizado vs Vectorial

Es fundamental entender la diferencia entre estos dos tipos de imágenes para elegir el formato correcto.

Rasterizado (mapa de bits)

Formado por una cuadrícula de píxeles. Cada píxel tiene un color.

  • Al ampliar se pixela (pierde calidad)
  • Tamaño del archivo proporcional a las dimensiones
  • Ideal para fotografías e imágenes complejas
  • Formatos: JPG, PNG, GIF, WebP, AVIF

Vectorial (matemático)

Definido por fórmulas matemáticas (puntos, líneas, curvas).

  • Escalable infinitamente sin perder calidad
  • Archivo ligero para formas geométricas
  • Ideal para logos, iconos, ilustraciones
  • Formato principal: SVG

Ejemplo visual

Rasterizado (JPG 100x100) ampliado a 400x400:
████    ← Se ven los píxeles (borroso)
████

Vectorial (SVG) ampliado a 400x400:
◯         ← Se ve nítido (recalcula las curvas)

<!-- Por eso los logos SIEMPRE deben ser SVG -->

6.4Imágenes responsive: <picture> y srcset

HTML ofrece herramientas para servir diferentes imágenes según el dispositivo: tamaño de pantalla, densidad de píxeles o soporte de formatos.

srcset — Diferentes resoluciones

El atributo srcset permite ofrecer varias versiones de la misma imagen a distintas resoluciones. El navegador elige la más adecuada. 

<!-- Por densidad de píxeles (x) -->
<img
  src="foto-400.jpg"
  srcset="foto-400.jpg 1x,
          foto-800.jpg 2x,
          foto-1200.jpg 3x"
  alt="Foto del equipo"
  width="400" height="300"
>

<!-- Por ancho de imagen (w) + sizes -->
<img
  src="foto-400.jpg"
  srcset="foto-400.jpg 400w,
          foto-800.jpg 800w,
          foto-1200.jpg 1200w"
  sizes="(max-width: 600px) 100vw,
         (max-width: 1200px) 50vw,
         33vw"
  alt="Foto del equipo"
>

¿Cómo funciona sizes?

  • (max-width: 600px) 100vw → en pantallas ≤600px, la imagen ocupa el 100% del ancho
  • (max-width: 1200px) 50vw → en pantallas ≤1200px, ocupa el 50%
  • 33vw → en el resto, ocupa un tercio
  • El navegador combina sizes con srcset para elegir la imagen óptima

<picture> — Formatos y art direction

El elemento <picture> permite ofrecer diferentes formatos (con fallback) y diferentes imágenes según media queries (art direction).

Fallback de formatos

<picture>
  <!-- AVIF: máxima compresión -->
  <source
    srcset="foto.avif"
    type="image/avif">

  <!-- WebP: buena compresión -->
  <source
    srcset="foto.webp"
    type="image/webp">

  <!-- JPG: fallback universal -->
  <img src="foto.jpg"
    alt="Foto del equipo"
    width="800" height="600">
</picture>

Art direction (distinta imagen)

<picture>
  <!-- Móvil: recorte vertical -->
  <source
    media="(max-width: 768px)"
    srcset="foto-movil.jpg">

  <!-- Tablet: recorte cuadrado -->
  <source
    media="(max-width: 1024px)"
    srcset="foto-tablet.jpg">

  <!-- Desktop: imagen completa -->
  <img src="foto-desktop.jpg"
    alt="Paisaje panorámico"
    width="1200" height="600">
</picture>

6.5<figure> y <figcaption>

El elemento <figure> agrupa contenido autoexplicativo (imagen, diagrama, código...) con su pie de descripción <figcaption>. Es semántico y mejora la accesibilidad.

Código

<figure>
  <img
    src="grafico-ventas.png"
    alt="Gráfico de barras mostrando
         ventas trimestrales 2024"
    width="600" height="400">

  <figcaption>
    Fig. 1 — Ventas trimestrales 2024.
    Fuente: Departamento de ventas.
  </figcaption>
</figure>

<!-- También funciona con código -->
<figure>
  <pre><code>console.log('Hola');</code></pre>
  <figcaption>Ejemplo en JavaScript</figcaption>
</figure>

Resultado visual

Nota: figcaption es complementario a alt, no lo sustituye. Ambos son necesarios.

6.6SVG inline en HTML

Los SVG se pueden incrustar directamente en el HTML. Esto permite estilizar con CSS y manipular con JavaScript cada parte del gráfico.

Como <img> (estático)

<!-- No se puede estilizar con CSS externo -->
<img src="icono.svg" alt="Icono"
  width="24" height="24">

Simple, cacheable, pero sin control CSS/JS.

Inline (manipulable)

<!-- Control total con CSS y JS -->
<svg width="24" height="24"
  viewBox="0 0 24 24"
  fill="none" xmlns="...">
  <circle cx="12" cy="12" r="10"
    stroke="currentColor"
    stroke-width="2"/>
  <path d="M12 6v6l4 2"
    stroke="currentColor"/>
</svg>

Permite cambiar colores, animar, interactuar.

¿Cuándo usar cada método?

Método Ventajas Inconvenientes
<img src="*.svg"> Cacheable, HTML limpio Sin control CSS/JS sobre partes internas
SVG inline Estilizable, animable, interactivo Ensucia el HTML, no cacheable como imagen
CSS background Decorativo, no ocupa DOM Sin accesibilidad (no es contenido)

6.7Mapas de imagen (<map>)

Un mapa de imagen permite definir zonas clicables dentro de una imagen. Cada zona (<area>) funciona como un enlace independiente. 

<!-- La imagen referencia al mapa con usemap -->
<img src="plano.jpg" alt="Plano de la oficina"
  usemap="#mapa-oficina" width="600" height="400">

<map name="mapa-oficina">
  <!-- Rectángulo: x1,y1,x2,y2 -->
  <area shape="rect" coords="0,0,200,150"
    href="sala-reuniones.html" alt="Sala de reuniones">

  <!-- Círculo: centroX,centroY,radio -->
  <area shape="circle" coords="300,200,50"
    href="recepcion.html" alt="Recepción">

  <!-- Polígono: x1,y1,x2,y2,x3,y3,... -->
  <area shape="poly" coords="400,100,500,50,600,100,500,200"
    href="despacho.html" alt="Despacho director">
</map>

Formas disponibles en <area>

shape coords Descripción
rect x1,y1,x2,y2 Rectángulo (esquina sup-izq, esquina inf-der)
circle cx,cy,radio Círculo (centro y radio en píxeles)
poly x1,y1,x2,y2,... Polígono (pares de coordenadas de vértices)

Nota: Los mapas de imagen no son responsive por defecto. Para webs modernas se suele preferir posicionar enlaces con CSS sobre la imagen.

6.8Favicon e imágenes especiales

Además de las imágenes visibles en la página, HTML utiliza imágenes para favicons, Open Graph (redes sociales) y pantallas de inicio (móviles).

Favicon (icono de pestaña)

<head>
  <!-- Favicon clásico (ICO, 32x32) -->
  <link rel="icon" href="/favicon.ico">

  <!-- Favicon moderno (SVG, escalable) -->
  <link rel="icon" href="/icon.svg" type="image/svg+xml">

  <!-- Apple Touch Icon (180x180 PNG) -->
  <link rel="apple-touch-icon" href="/apple-touch-icon.png">

  <!-- Iconos para diferentes tamaños -->
  <link rel="icon" type="image/png" sizes="32x32"
    href="/favicon-32x32.png">
  <link rel="icon" type="image/png" sizes="16x16"
    href="/favicon-16x16.png">
</head>

Open Graph (redes sociales)

<head>
  <!-- Imagen al compartir en redes sociales -->
  <meta property="og:image"
    content="https://miweb.com/img/og-image.jpg">
  <meta property="og:image:width" content="1200">
  <meta property="og:image:height" content="630">
  <meta property="og:image:alt"
    content="Descripción de la imagen">

  <!-- Twitter Card (imagen grande) -->
  <meta name="twitter:card"
    content="summary_large_image">
  <meta name="twitter:image"
    content="https://miweb.com/img/twitter-card.jpg">
</head>

Tamaño recomendado OG: 1200×630px. Twitter Card: 1200×675px.

6.9Optimización y rendimiento

Las imágenes suelen ser el recurso más pesado de una web. Optimizarlas impacta directamente en el tiempo de carga y en los Core Web Vitals (LCP, CLS).

Lazy loading (carga diferida)

Solo carga la imagen cuando está a punto de entrar en el viewport. Reduce el tiempo de carga inicial.

<!-- Nativo en HTML: sin JavaScript -->
<img src="foto.jpg" alt="Foto"
  loading="lazy"
  width="600" height="400">

<!-- NO usar lazy en imágenes above the fold (hero, logo...) -->
<img src="hero.webp" alt="Banner principal"
  loading="eager"
  fetchpriority="high">

Evitar Layout Shift (CLS)

Si no defines width y height, la página “salta” cuando la imagen se carga.

Mal — causa CLS

<img src="foto.jpg" alt="Foto">
<!-- Sin dimensiones: el navegador
     no reserva espacio -->

Bien — sin CLS

<img src="foto.jpg" alt="Foto"
  width="600" height="400">
<!-- O con CSS aspect-ratio -->

Preload de imágenes críticas

<head>
  <!-- Prioriza la descarga del hero image -->
  <link rel="preload" as="image"
    href="hero.webp" type="image/webp">

  <!-- Con srcset para responsive -->
  <link rel="preload" as="image"
    href="hero-800.webp"
    imagesrcset="hero-400.webp 400w,
               hero-800.webp 800w,
               hero-1200.webp 1200w"
    imagesizes="100vw">
</head>

Checklist de optimización de imágenes

  • Usar formatos modernos (WebP/AVIF) con fallback
  • Siempre definir width y height
  • loading="lazy" en imágenes below the fold
  • fetchpriority="high" en el hero / LCP image
  • Usar srcset + sizes para servir tamaños adecuados
  • Comprimir imágenes antes de subir (TinyPNG, Squoosh)
  • No servir imágenes mayores de lo que se muestran
  • SVG para logos e iconos
  • Alt descriptivo en todas las imágenes
  • Preload para la imagen LCP (hero, banner)
