10Formularios

Los formularios son el principal medio para recoger información del usuario en una página web. Permiten enviar datos al servidor para su procesamiento.

  • <form>: contenedor principal que agrupa todos los elementos del formulario
  • <input>: campo de entrada de datos (el más versátil)
  • <label>: etiqueta descriptiva asociada al input (mejora accesibilidad)
  • <textarea>: área de texto multilínea
  • <select> + <option>: lista desplegable de opciones
  • <button>: botón para enviar o realizar acciones
  • <fieldset> + <legend>: agrupa campos relacionados
  • <datalist>: proporciona sugerencias de autocompletado

Importante: Siempre usa <label> con el atributo for que coincida con el id del input para mejorar la accesibilidad y permitir hacer clic en la etiqueta.

11Atributos del <form>

El elemento <form> tiene atributos esenciales que definen cómo se envían los datos.

Atributo Descripción Valores
action URL donde se envían los datos URL o ruta relativa
method Método HTTP de envío GET (defecto) | POST
enctype Tipo de codificación de datos Ver nota abajo
target Dónde mostrar la respuesta _self | _blank
autocomplete Habilitar autocompletado del navegador on | off
novalidate Desactiva validación HTML5 Booleano (sin valor)

Valores de enctype:

  • application/x-www-form-urlencoded: por defecto, codifica espacios como +
  • multipart/form-data: obligatorio para subir archivos
  • text/plain: sin codificación (poco usado)
<form action="/enviar" method="POST" enctype="multipart/form-data">
    <input type="file" name="archivo">
    <button type="submit">Subir</button>
</form>

12Tipos de Input

El elemento <input> es el más versátil de los formularios. El atributo type define su comportamiento y apariencia.

Inputs de Texto

Tipo Descripción Ejemplo
text Texto de una línea (por defecto)
password Texto oculto con puntos/asteriscos
email Valida formato de email. Teclado optimizado en móviles
tel Teléfono. No valida formato pero muestra teclado numérico
url Valida formato de URL completa
search Campo de búsqueda (puede mostrar X para borrar)

Inputs Numéricos

Tipo Descripción Ejemplo
number Números con flechas +/-. Usa min, max, step
range Deslizador para seleccionar valor en rango

Inputs de Fecha y Hora

Tipo Descripción Ejemplo
date Selector de fecha (día/mes/año)
time Selector de hora (HH:MM)
datetime-local Fecha y hora combinadas (sin zona horaria)
month Selector de mes y año
week Selector de semana del año

Inputs de Selección

Tipo Descripción Ejemplo
checkbox Casilla de verificación (selección múltiple)
radio Botón circular (selección única, usar mismo name)

Otros Inputs Especiales

Tipo Descripción Ejemplo
file Selector de archivos. Usa accept para filtrar tipos y multiple para varios
color Selector de color con paleta
hidden Campo oculto (envía datos sin mostrar al usuario) No visible

13Textarea

El elemento <textarea> permite al usuario escribir texto en múltiples líneas. A diferencia de <input>, tiene etiqueta de cierre.

Atributos principales

  • rows: número de filas visibles (alto)
  • cols: número de columnas visibles (ancho)
  • placeholder: texto de ayuda
  • maxlength: máximo de caracteres permitidos
  • minlength: mínimo de caracteres requeridos
  • wrap: cómo se envían los saltos de línea (soft|hard)
  • readonly: solo lectura
  • disabled: deshabilitado
<label for="mensaje">Tu mensaje:</label>
<textarea id="mensaje" name="mensaje"
    rows="4" cols="50"
    placeholder="Escribe aquí..."
    maxlength="500"></textarea>

Máximo 200 caracteres

CSS resize: Usa resize: none|vertical|horizontal|both en CSS para controlar si el usuario puede redimensionar el textarea.

14Select y Datalist

<select> - Lista desplegable

Permite elegir una o varias opciones de una lista predefinida.

  • <option>: cada opción. Usa value para el valor enviado
  • <optgroup>: agrupa opciones con etiqueta label
  • multiple: permite selección múltiple (con Ctrl/Cmd)
  • size: número de opciones visibles sin desplegar
  • selected: opción preseleccionada

<datalist> - Autocompletado

Proporciona sugerencias mientras el usuario escribe, pero permite valores libres.

<input type="text" list="navegadores" placeholder="Elige o escribe...">
<datalist id="navegadores">
    <option value="Chrome">
    <option value="Firefox">
    <option value="Safari">
    <option value="Edge">
</datalist>

Diferencia clave: <select> fuerza a elegir entre opciones. <datalist> sugiere pero permite escribir cualquier valor.

15Atributos de Input

Los inputs tienen numerosos atributos para controlar su comportamiento.

Atributos esenciales

Atributo Descripción
name Obligatorio para enviar datos. Clave en el par clave=valor
id Identificador único para asociar con <label for>
value Valor inicial o actual del campo
placeholder Texto de ayuda que desaparece al escribir

Atributos de estado

Atributo Efecto Demo
disabled Deshabilitado. No se envía con el formulario
readonly Solo lectura. Sí se envía con el formulario
autofocus Recibe foco automáticamente al cargar Solo 1 por página
checked Preseleccionado (solo checkbox/radio) marcado

Atributos para archivos

  • accept: tipos MIME permitidos (image/*, .pdf, video/*)
  • multiple: permite seleccionar varios archivos
  • capture: en móviles, abre cámara (user|environment)

16Fieldset y Legend

Estos elementos agrupan campos relacionados, mejorando la organización y accesibilidad del formulario.

  • <fieldset>: agrupa controles relacionados con un borde visual
  • <legend>: título del grupo (aparece sobre el borde)
<fieldset>
    <legend>Datos personales</legend>
    <label>Nombre: <input type="text" name="nombre"></label>
    <label>Email: <input type="email" name="email"></label>
</fieldset>

17Validación de Formularios

HTML5 incluye validación nativa sin necesidad de JavaScript. El navegador muestra mensajes de error automáticamente.

Atributos de validación

Atributo Descripción Ejemplo
required Campo obligatorio <input required>
minlength Mínimo de caracteres minlength="3"
maxlength Máximo de caracteres maxlength="50"
min / max Valor mínimo/máximo (números, fechas) min="0" max="100"
step Incremento permitido step="0.01" (decimales)
pattern Expresión regular para validar pattern="[A-Za-z]+"

Ejemplos prácticos de pattern

Uso Pattern
Solo letras [A-Za-záéíóúÁÉÍÓÚñÑ]+
Teléfono español [679][0-9]{8}
Código postal [0-9]{5}
Contraseña segura (?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{8,}

Prueba la validación

Pseudoclases CSS para validación:

  • :valid - campo válido
  • :invalid - campo inválido
  • :required - campo obligatorio
  • :optional - campo opcional
  • :in-range / :out-of-range - para valores numéricos

18Botones

Hay dos formas de crear botones: <button> e <input type="...">.

Tipos de <button>

  • type="submit": envía el formulario (por defecto)
  • type="reset": restablece todos los campos a su valor inicial
  • type="button": botón genérico (no hace nada sin JS)

<button> vs <input type="submit">

<button> <input type="submit">
Puede contener HTML (iconos, imágenes) Solo texto plano (atributo value)
Más flexible para estilizar Estilo limitado
Recomendado Uso legacy

19Asociación Label-Input

La correcta asociación entre <label> e <input> es fundamental para la accesibilidad y usabilidad de los formularios.

¿Por qué es importante?

  • Accesibilidad: Los lectores de pantalla anuncian el label cuando el input recibe el foco
  • Usabilidad: Al hacer clic en el label, el input recibe el foco automáticamente
  • Área de clic ampliada: Especialmente útil para checkboxes y radios pequeños
  • Semántica: Indica claramente la relación entre texto descriptivo y campo

Método 1: Asociación explícita (for/id)

El atributo for del label debe coincidir exactamente con el id del input.

<!-- Asociación explícita: for="X" debe coincidir con id="X" -->
<label for="email">Correo electrónico:</label>
<input type="email" id="email" name="email">

<!-- Pueden estar separados en el DOM -->
<div class="campo">
    <label for="nombre">Nombre completo:</label>
</div>
<div class="input">
    <input type="text" id="nombre" name="nombre">
</div>

Ventaja: Más flexible, permite posicionar label e input en cualquier lugar del HTML. Método recomendado.

Método 2: Asociación implícita (anidamiento)

El input se coloca dentro del label. No requiere for/id.

<!-- Asociación implícita: input dentro del label -->
<label>
    Nombre de usuario:
    <input type="text" name="username">
</label>

<!-- Muy común para checkboxes y radios -->
<label>
    <input type="checkbox" name="terminos">
    Acepto los términos y condiciones
</label>

Nota: Algunos lectores de pantalla antiguos pueden tener problemas con este método. Por seguridad, combina ambos:

<label for="check1"><input type="checkbox" id="check1"> Texto</label>

20Formularios Semánticos

Un formulario semántico usa los elementos HTML correctos para su propósito, mejorando la accesibilidad, SEO y mantenibilidad del código.

Estructura semántica completa

<form action="/registro" method="POST">
    <!-- Agrupación con fieldset y legend -->
    <fieldset>
        <legend>Información personal</legend>

        <!-- Campo con label explícito -->
        <div class="campo">
            <label for="nombre">
                Nombre completo
                <span class="requerido" aria-hidden="true">*</span>
            </label>
            <input type="text" id="nombre" name="nombre"
                required
                autocomplete="name"
                aria-describedby="nombre-ayuda">
            <span id="nombre-ayuda" class="ayuda">
                Introduce tu nombre y apellidos
            </span>
        </div>

        <!-- Campo de email con autocomplete -->
        <div class="campo">
            <label for="email">Correo electrónico</label>
            <input type="email" id="email" name="email"
                required
                autocomplete="email"
                placeholder="ejemplo@dominio.com">
        </div>
    </fieldset>

    <fieldset>
        <legend>Preferencias</legend>

        <!-- Grupo de radio buttons -->
        <div role="radiogroup" aria-labelledby="contacto-label">
            <span id="contacto-label">Contactar por:</span>
            <label>
                <input type="radio" name="contacto" value="email" checked>
                Email
            </label>
            <label>
                <input type="radio" name="contacto" value="telefono">
                Teléfono
            </label>
        </div>
    </fieldset>

    <button type="submit">Enviar formulario</button>
</form>

Atributo autocomplete

Indica al navegador qué tipo de dato espera el campo, permitiendo autocompletado inteligente:

Valor Dato esperado Valor Dato esperado
name Nombre completo email Email
given-name Nombre tel Teléfono
family-name Apellidos street-address Dirección
username Usuario postal-code Código postal
new-password Nueva contraseña country País
current-password Contraseña actual bday Fecha nacimiento

21Accesibilidad Avanzada (ARIA)

Los atributos ARIA (Accessible Rich Internet Applications) mejoran la accesibilidad cuando el HTML semántico no es suficiente.

Atributos ARIA para formularios

Atributo Uso Ejemplo
aria-label Etiqueta invisible para lectores de pantalla aria-label="Buscar productos"
aria-labelledby Referencia a elemento que actúa como label aria-labelledby="titulo-seccion"
aria-describedby Referencia a descripción adicional aria-describedby="ayuda-pass"
aria-required Indica campo obligatorio (complementa required) aria-required="true"
aria-invalid Indica que el valor es inválido aria-invalid="true"
aria-errormessage Referencia al mensaje de error aria-errormessage="error-email"
aria-hidden Oculta elemento decorativo del lector aria-hidden="true" (para iconos, *)
role Define el rol del elemento role="alert", role="group"

Ejemplo: Campo con validación accesible

<div class="campo">
    <label for="password">
        Contraseña
        <span class="requerido" aria-hidden="true">*</span>
    </label>

    <input type="password" id="password" name="password"
        required
        minlength="8"
        aria-required="true"
        aria-describedby="password-requisitos"
        aria-invalid="false"
        aria-errormessage="password-error">

    <!-- Instrucciones siempre visibles -->
    <p id="password-requisitos" class="ayuda">
        Mínimo 8 caracteres, incluir mayúsculas y números
    </p>

    <!-- Error solo visible cuando aplica -->
    <p id="password-error" class="error" role="alert" hidden>
        La contraseña no cumple los requisitos
    </p>
</div>

Buenas prácticas de accesibilidad

✔ HAZ

  • Usa <label> para TODOS los campos
  • Agrupa campos relacionados con <fieldset>
  • Proporciona mensajes de error claros y específicos
  • Permite navegación completa por teclado
  • Indica visualmente el foco actual
  • Usa autocomplete apropiado
  • Mantiene contraste mínimo 4.5:1

✘ EVITA

  • Usar solo placeholder como etiqueta
  • Depender solo del color para indicar errores
  • Usar tabindex positivo (altera orden natural)
  • Deshabilitar el zoom en móviles
  • Usar CAPTCHA sin alternativa accesible
  • Límites de tiempo sin opción de extensión
  • Auto-avanzar entre campos sin avisar

Recuerda: La primera regla de ARIA es "No uses ARIA si puedes usar HTML semántico nativo". Un <button> es mejor que <div role="button">.
