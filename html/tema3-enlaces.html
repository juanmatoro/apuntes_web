5El elemento <a>

El elemento <a> (anchor) crea hipervínculos, la base de la navegación web. Permite conectar documentos, secciones, archivos y recursos externos.

Atributos principales

  • href — URL de destino (obligatorio para que sea un enlace)
  • target — dónde abrir el enlace (_self, _blank, _parent, _top)
  • rel — relación con el documento destino (seguridad, SEO)
  • download — fuerza descarga en lugar de navegar
  • hreflang — idioma del documento destino
  • type — tipo MIME del recurso enlazado
  • title — información adicional (tooltip)
<!-- Sintaxis básica -->
<a href="destino.html">Texto del enlace</a>

<!-- Enlace completo con atributos -->
<a href="https://ejemplo.com/documento.pdf"
   target="_blank"
   rel="noopener noreferrer"
   download="mi-documento.pdf"
   title="Descargar documento PDF">
    Descargar PDF
</a>

Nota: Un <a> sin href es válido pero no es un enlace — actúa como marcador de posición o ancla.

Rutas relativas

<!-- Mismo directorio -->
<a href="pagina.html">Misma carpeta</a>

<!-- Subdirectorio -->
<a href="carpeta/pagina.html">Dentro de carpeta</a>

<a href="../pagina.html">Un nivel arriba</a>

<!-- Dos niveles arriba -->
<a href="../../pagina.html">Dos niveles arriba</a>

<!-- Desde la raíz del sitio -->
<a href="/carpeta/pagina.html">Desde raíz</a>

7Atributo target

El atributo target especifica dónde se abrirá el enlace.

Valor Comportamiento
_self Misma pestaña/ventana (por defecto)
_blank Nueva pestaña o ventana
_parent Marco padre (en iframes anidados)
_top Ventana completa (sale de todos los iframes)
nombre iframe o ventana con ese nombre 
<!-- Nueva pestaña (SIEMPRE con rel="noopener") -->
<a href="https://ejemplo.com"
   target="_blank"
   rel="noopener noreferrer">
    Abrir en nueva pestaña
</a>

<!-- Abrir en un iframe específico -->
<a href="contenido.html" target="mi-iframe">
    Cargar en iframe
</a>
<iframe name="mi-iframe"></iframe>

Seguridad: Usar target="_blank" sin rel="noopener" es una vulnerabilidad. La página destino puede acceder a window.opener y manipular tu página.

8Atributo rel

El atributo rel define la relación entre el documento actual y el destino. Es crucial para seguridad y SEO.

Valor Descripción Uso
noopener Impide acceso a window.opener Seguridad
noreferrer No envía cabecera Referer Privacidad
nofollow No seguir para SEO (no pasar PageRank) SEO
sponsored Enlace patrocinado/publicitario SEO
ugc Contenido generado por usuarios SEO
external Indica enlace externo Semántico
author Enlace al autor del documento Semántico
license Enlace a licencia del contenido Semántico
help Enlace a ayuda Semántico
prev / next Navegación secuencial Navegación
bookmark Enlace permanente al contenido Semántico 
<!-- Enlace externo seguro (OBLIGATORIO con target="_blank") -->
<a href="https://externo.com" target="_blank"
   rel="noopener noreferrer">
    Sitio externo
</a>

<!-- Enlace patrocinado (publicidad) -->
<a href="https://sponsor.com"
   rel="sponsored noopener" target="_blank">
    Patrocinador
</a>

<!-- Comentario de usuario (UGC) -->
<a href="https://usuario.com"
   rel="ugc nofollow noopener" target="_blank">
    Enlace de comentario
</a>

<!-- Navegación secuencial -->
<a href="cap-1.html" rel="prev">← Anterior</a>
<a href="cap-3.html" rel="next">Siguiente →</a>

Se pueden combinar: rel="noopener noreferrer nofollow" aplica los tres a la vez.

9Enlaces de descarga

El atributo download indica al navegador que debe descargar el recurso en lugar de navegar a él. 

<!-- Descarga con nombre original -->
<a href="/archivos/documento.pdf" download>
    Descargar PDF
</a>

<!-- Descarga con nombre personalizado -->
<a href="/archivos/doc-v2.pdf"
   download="informe-anual-2024.pdf">
    Descargar Informe 2024
</a>

<!-- Descargar imagen -->
<a href="/img/logo.png" download="logo-empresa.png">
    Descargar logo
</a>

<!-- Descargar con tipo MIME -->
<a href="/data/export.csv"
   download="datos.csv"
   type="text/csv">
    Exportar CSV
</a>

Funciona

  • Archivos del mismo origen (same-origin)
  • URLs blob:
  • URLs data:

No funciona

  • Archivos de otro dominio (cross-origin) sin CORS
  • El servidor debe permitir la descarga

10Estados de enlaces (CSS)

Los enlaces tienen estados que se pueden estilizar con pseudoclases CSS. El orden importa (LVHA).

Pseudoclase Estado Descripción
:link No visitado Enlace que aún no se ha visitado
:visited Visitado Enlace ya visitado (limitado por privacidad)
:hover Hover Cursor encima del enlace
:active Activo Mientras se hace clic
:focus Foco Seleccionado con teclado (Tab)
:focus-visible Foco visible Foco solo cuando es relevante mostrar 
/* Orden recomendado: LVHA (LoVe HAte) */
a:link {
    color: blue;
}

a:visited {
    color: purple;
    /* Limitado a: color, background-color,
       border-color, outline-color */
}

a:hover {
    color: darkblue;
    text-decoration: underline;
}

a:active {
    color: red;
}

a:focus {
    outline: 2px solid orange;
    outline-offset: 2px;
}

/* focus-visible: solo muestra outline con teclado */
a:focus-visible {
    outline: 2px solid blue;
}

a:focus:not(:focus-visible) {
    outline: none; /* Oculta con mouse */
}

11Seguridad en enlaces

Vulnerabilidad: Tabnabbing

Cuando usas target="_blank" sin protección, la página destino puede acceder a window.opener y redirigir tu página a un sitio de phishing.

✘ Vulnerable

<a href="https://malicioso.com"
   target="_blank">
    Enlace peligroso
</a>

// El sitio malicioso puede hacer:
window.opener.location = "https://phishing.com"

✔ Seguro

<a href="https://externo.com"
   target="_blank"
   rel="noopener noreferrer">
    Enlace seguro
</a>

// window.opener será null
// No se envía Referer

Reglas de seguridad

  • Siempre añade rel="noopener" con target="_blank"
  • Añade noreferrer si no quieres compartir el origen
  • Usa nofollow para enlaces no confiables (comentarios, UGC)
  • Valida URLs de usuarios antes de mostrarlas
  • Nunca uses javascript: en href con datos de usuario

Navegadores modernos: Chrome 88+ y otros navegadores modernos aplican noopener automáticamente a target="_blank", pero sigue siendo buena práctica añadirlo explícitamente para compatibilidad.

12Accesibilidad en enlaces

Texto descriptivo

El texto del enlace debe describir su destino sin depender del contexto.

✘ Mal

  • Clic aquí
  • Leer más
  • Enlace
  • https://ejemplo.com/articulo/123

✔ Bien

  • Ver guía de instalación
  • Leer artículo sobre accesibilidad
  • Descargar informe 2024 (PDF)
  • Documentación de la API

Skip links (saltar navegación)

Permite a usuarios de teclado/lectores saltar directamente al contenido principal.

<body>
    <!-- Primer elemento del body -->
    <a href="#main" class="skip-link">
        Saltar al contenido principal
    </a>

    <header>...navegación...</header>

    <main id="main">
        <!-- Contenido -->
    </main>
</body>

<style>
.skip-link {
    position: absolute;
    top: -40px;
    left: 0;
    padding: 8px;
    background: #000;
    color: #fff;
    z-index: 100;
}
.skip-link:focus {
    top: 0; /* Visible al hacer Tab */
}
</style>

ARIA para enlaces

<!-- Indicar que abre en nueva pestaña -->
<a href="..." target="_blank"
   aria-label="Documentación (abre en nueva pestaña)">
    Documentación <span aria-hidden="true">↗</span>
</a>

<!-- Enlace con descripción adicional -->
<a href="..." aria-describedby="desc-pdf">
    Descargar informe
</a>
<span id="desc-pdf" class="sr-only">
    Archivo PDF, 2.5 MB
</span>

<!-- Enlace que es el único propósito de su contenedor -->
<article>
    <h2><a href="...">Título del artículo</a></h2>
    <p>Resumen...</p>
    <a href="..." aria-label="Leer: Título del artículo">
        Leer más
    </a>
</article>

Checklist de accesibilidad

  • Texto descriptivo que tenga sentido fuera de contexto
  • Indicar visualmente si abre en nueva pestaña
  • Indicar tipo de archivo y tamaño en descargas
  • Foco visible al navegar con teclado
  • Contraste suficiente (4.5:1 mínimo)
  • No usar solo color para diferenciar enlaces del texto
  • Skip link al inicio de la página

13Buenas prácticas

✔ HAZ

  • Texto descriptivo y significativo
  • rel="noopener" con target="_blank"
  • Indicar tipo de archivo en descargas
  • Diferenciar visualmente enlaces de texto
  • Asegurar foco visible con teclado
  • Usar rutas relativas para enlaces internos
  • Incluir skip link para accesibilidad
  • Verificar enlaces rotos regularmente

✘ EVITA

  • “Clic aquí”, “Leer más” sin contexto
  • javascript:void(0) como href
  • URLs largas como texto del enlace
  • Abrir todas las páginas en nueva pestaña
  • Enlaces que no parecen enlaces
  • Elementos no-enlace que actúan como enlaces
  • Eliminar el outline de foco sin alternativa
  • Enlaces vacíos o con solo imagen sin alt

Patrón recomendado para enlaces externos

<!-- Plantilla para enlaces externos -->
<a href="https://sitio-externo.com"
   target="_blank"
   rel="noopener noreferrer"
   class="external-link">
    Nombre descriptivo del destino
    <span class="icon" aria-hidden="true">↗</span>
    <span class="sr-only">(abre en nueva pestaña)</span>
</a>

<!-- Plantilla para descargas -->

   download="nombre-descriptivo.pdf"
   class="download-link">
    Descargar guía completa
    <span class="meta">(PDF, 2.5 MB)</span>
</a>
