1El modelo de caja

Todo elemento HTML es interpretado por el navegador como una caja rectangular compuesta por 4 áreas concéntricas:

Margin (Externo)
Border (Límite)
Padding (Interno)
Content

La analogía del metro: El Margin es tu espacio personal para no tocar a otros; el Padding es como llevar una chaqueta gruesa: te aleja de la superficie del borde, pero te hace ver más grande hacia afuera.

Anatomía de la Caja

  • Content: El área donde vive el texto o imágenes. Sus dimensiones se definen con width e height.
  • Padding: Espacio dentro del borde. Es transparente pero muestra el color de fondo del elemento.
  • Border: La línea que envuelve el padding y el contenido. Tiene grosor, estilo y color.
  • Margin: Espacio fuera del borde. Separa al elemento de sus vecinos y es siempre transparente.

El problema del tamaño real

Por defecto (content-box), el padding y el borde se suman al ancho definido. Si quieres que tu caja mida exactamente lo que indicas, debes usar:

box-sizing: border-box;

Sintaxis y Propiedades

.box {
    /* 1. Dimensiones base */
    width: 200px; 
    height: 100px;

    /* 2. Respiración interna (Chaqueta) */
    padding: 20px;               /* Arriba, Derecha, Abajo, Izquierda */
    padding-block: 10px;         /* Lógico: Arriba y Abajo */
    padding-inline: 20px;        /* Lógico: Izquierda y Derecha */

    /* 3. Marco visual */
    border: 2px solid #333;

    /* 4. Espacio personal (Separación) */
    margin: 0 auto;              /* Centra bloques con ancho definido */
}

Tipos de caja según display

La propiedad display determina cómo se comporta la caja en el flujo del documento:

block

Ocupa todo el ancho. Permite modificar todo el Box Model.

inline

Fluye con el texto. No acepta width/height ni márgenes verticales.

inline-block

Híbrido: fluye en línea pero permite redimensionar la caja.

2box-sizing

Controla cómo se calcula el tamaño total del elemento.

content-box (default)

width/height solo incluyen el contenido. Padding y border se suman. 

.box {
    width: 200px;
    padding: 20px;
    border: 5px solid;
    /* Ancho total: 200 + 40 + 10 = 250px */
}

border-box (recomendado)

width/height incluyen contenido + padding + border. 

.box {
    box-sizing: border-box;
    width: 200px;
    padding: 20px;
    border: 5px solid;
    /* Ancho total: 200px (siempre) */
}

/* Reset universal recomendado */
*, *::before, *::after {
    box-sizing: border-box;
}

3Dimensiones

Ancho y alto

.box {
    width: 300px;
    height: 200px;

    /* Límites */
    min-width: 100px;
    max-width: 800px;
    min-height: 50px;
    max-height: 500px;
}

Comportamiento por defecto

  • Block: width 100%, height auto (según contenido)
  • Inline: width y height según contenido (no se pueden cambiar)
  • Inline-block: width/height modificables, flujo inline

Margin auto

/* Centrar horizontalmente un bloque */
.centered {
    width: 80%;
    margin-inline: auto;
}

/* Margin negativo */
.overlap {
    margin-top: -20px;
}

4aspect-ratio

2020 - Define la proporción ancho/alto de un elemento. 

.video-container {
    width: 100%;
    aspect-ratio: 16 / 9;  /* Proporción de video HD */
}

.square {
    width: 200px;
    aspect-ratio: 1;       /* 1:1 = cuadrado */
}

.portrait {
    aspect-ratio: 3 / 4;   /* Tipo retrato */
}

/* Con iframes */
iframe {
    width: 100%;
    aspect-ratio: 16 / 9;
    height: auto;
}

Tip: Si defines width Y height explícitos, aspect-ratio se ignora. Define solo uno de los dos.

5object-fit y object-position

Controla cómo el contenido de <img>, <video> o <iframe> se ajusta a su contenedor.

object-fit

  • fill - Estira para llenar (puede distorsionar)
  • contain - Cabe entero, puede dejar espacio
  • cover - Cubre todo, puede recortar
  • none - Tamaño original
  • scale-down - El menor entre none y contain 
.thumbnail {
    width: 150px;
    height: 150px;
    object-fit: cover;  /* Recorta para llenar */
}

.logo {
    width: 200px;
    height: 100px;
    object-fit: contain;  /* Cabe sin distorsionar */
}

object-position

.avatar {
    object-fit: cover;
    object-position: center top;  /* Enfoca en la cara */
}

.product {
    object-fit: contain;
    object-position: center center;
}

6Sizing keywords

Valores especiales para width/height que se adaptan al contenido.

min-content

El tamaño mínimo para que quepa el contenido (palabra más larga). 

.tag {
    width: min-content;  /* Se ajusta a la palabra más larga */
}

max-content

El tamaño ideal del contenido sin wrapping. 

.title {
    width: max-content;  /* Todo el texto en una línea */
}

fit-content

Entre min-content y max-content, con un límite opcional. 

.card {
    width: fit-content;        /* Ajusta al contenido */
    max-width: 100%;           /* Pero no excede el padre */
}

.limited {
    width: fit-content(300px); /* Máximo 300px */
}

Uso común: Botones, etiquetas y elementos que deben ajustarse a su contenido pero no crecer indefinidamente.

6Propiedades Lógicas (Logical Properties)

Es la evolución del Box Model. En lugar de usar direcciones físicas (arriba, abajo, izquierda, derecha), usamos direcciones basadas en el flujo del contenido.

Entendiendo los Ejes

Dependen del writing-mode (modo de escritura).

Eje Inline (En línea)

Dirección en la que viaja el texto al escribir. En español es Horizontal (de izquierda a derecha).

Eje Block (Bloque)

Dirección en la que se apilan los bloques (párrafos). En español es Vertical (de arriba hacia abajo).

Tabla de Equivalencias

Prop. Física Prop. Lógica
width inline-size
height block-size
margin-left margin-inline-start
margin-right margin-inline-end
padding-top padding-block-start
border-bottom border-block-end

Shorthands Modernos 

/* Afecta Izquierda y Derecha a la vez */
.caja {
  margin-inline: auto;
  padding-inline: 20px;
}

/* Afecta Arriba y Abajo a la vez */
.caja {
  margin-block: 1rem;
  padding-block: 2rem;
}

* Nota: Usar propiedades lógicas hace que tu sitio soporte idiomas como el Árabe (RTL) automáticamente.
