1Transiciones

Suavizan el cambio entre estados de un elemento.

Definición

Una transición es el “suavizado” entre dos estados: en vez de cambiar de golpe, la propiedad se mueve a su nuevo valor con un tiempo y un ritmo definidos.

Cuándo usarla

Ideal para feedback rápido: hover en botones, menús desplegables, cambios de foco, activar/desactivar toggles. En tiendas, va bien para elevar cards de producto o resaltar filtros al pasar el cursor.

Recomendaciones

Mantén duraciones cortas (100–300ms), usa transform y opacity cuando sea posible y evita transicionar propiedades que fuerzan reflow (width/height).

Propiedades

.button {
    transition-property: background, transform;
    transition-duration: 0.3s;
    transition-timing-function: ease-out;
    transition-delay: 0s;

    /* Atajo */
    transition: all 0.3s ease-out;
    transition: background 0.3s, transform 0.2s;
}

.button:hover {
    background: blue;
    transform: scale(1.05);
}

Definición de cada propiedad

  • transition-property: define qué propiedades cambian de forma gradual. Solo las que indiques (o all) se animan.
  • transition-duration: tiempo que tarda la transición en completarse, desde el valor inicial al final.
  • transition-timing-function: curva de velocidad del movimiento (cómo acelera y desacelera).
  • transition-delay: espera antes de comenzar la transición (no afecta a la duración).
  • transition (atajo): combina propiedad, duración, función y delay. Puedes encadenar varias transiciones separadas por comas.

Timing functions

transition-timing-function:
    ease;          /* Default, suave */
    linear;        /* Velocidad constante */
    ease-in;       /* Empieza lento */
    ease-out;      /* Termina lento */
    ease-in-out;   /* Lento al inicio y final */

    /* Personalizado */
    cubic-bezier(0.68, -0.55, 0.265, 1.55);

    /* Steps (discreto) */
    steps(4, end);

Qué hace cada valor

  • ease: aceleración suave al inicio y desaceleración al final; es el valor por defecto.
  • linear: mantiene la misma velocidad todo el tiempo (sin aceleraciones).
  • ease-in: empieza lento y va acelerando hacia el final.
  • ease-out: arranca rápido y termina lento.
  • ease-in-out: lento al inicio y al final, más rápido en el centro.
  • cubic-bezier(...): curva personalizada con 4 puntos de control para ajustar el ritmo con exactitud.
  • steps(n, end/start): movimiento a saltos en n pasos; útil para contadores o sprites.

Propiedades animables

No todas las propiedades pueden tener transición. Las más eficientes:

  • Recomendadas (GPU): transform, opacity
  • Otras: color, background-color, border-color, box-shadow, filter
  • Evitar: width, height, margin (causan reflow)

2Transformaciones

Definición

transform modifica un elemento sin afectar el flujo del documento: puedes moverlo, escalarlo, rotarlo o inclinarlo. Con 3D, se simula profundidad.

Cuándo usarla

Perfecta para micro-interacciones: tarjetas que “suben”, badges inclinados, iconos que rotan en un menú, o botones que se encogen al hacer click. En ecommerce, ayuda a destacar precios o promociones sin mover el layout.

Recomendaciones

Ajusta transform-origin para que las rotaciones se sientan naturales y oculta la cara trasera con backface-visibility: hidden en efectos de flip.

Funciones 2D

.element {
    /* Mover */
    transform: translate(10px, 20px);
    transform: translateX(10px);
    transform: translateY(20px);

    /* Escalar */
    transform: scale(1.5);
    transform: scale(1.5, 0.5);  /* X, Y */

    /* Rotar */
    transform: rotate(45deg);

    /* Sesgar */
    transform: skew(10deg, 5deg);
    transform: skewX(10deg);

    /* Múltiples */
    transform: translateX(50px) rotate(45deg) scale(1.2);
}

Funciones 3D

.element {
    transform: translate3d(10px, 20px, 30px);
    transform: translateZ(50px);

    transform: rotate3d(1, 1, 0, 45deg);
    transform: rotateX(45deg);
    transform: rotateY(45deg);
    transform: rotateZ(45deg);

    transform: scale3d(1, 1.5, 2);
}

/* Perspectiva */
.container {
    perspective: 1000px;  /* En el padre */
}
.card {
    transform: perspective(1000px) rotateY(45deg);
}

Propiedades auxiliares

.element {
    transform-origin: center center;  /* Default */
    transform-origin: top left;
    transform-origin: 50% 100%;

    transform-style: flat | preserve-3d;  /* Para 3D anidado */
    backface-visibility: visible | hidden;  /* Cara trasera */
}

Definición de propiedades

  • transform: aplica una o varias funciones (translate, scale, rotate, skew) en el orden escrito.
  • translate / translateX / translateY / translateZ: mueve el elemento en el eje indicado sin afectar el flujo del layout.
  • scale / scale3d: cambia el tamaño; valores > 1 agrandan, < 1 reducen.
  • rotate / rotateX / rotateY / rotateZ: gira el elemento alrededor del eje elegido.
  • skew / skewX / skewY: inclina el elemento en el eje indicado.
  • transform-origin: punto desde donde se aplican las transformaciones (por defecto el centro).
  • perspective: distancia de la cámara para crear profundidad 3D (normalmente en el padre).
  • transform-style: preserve-3d mantiene el 3D de hijos; flat lo aplana.
  • backface-visibility: controla si se ve la cara trasera cuando un elemento rota en 3D.

3Animaciones (@keyframes)

Definición

@keyframes describe pasos de una animación (del 0% al 100%). Luego animation decide cuánto dura, cuántas veces se repite y su ritmo.

Cuándo usarla

Útil para loaders, badges que “respiran”, skeletons suaves o para enfatizar un cambio de estado en cards de producto. En dashboards, sirve para animar indicadores o alertas sin ser invasivo.

Recomendaciones

No abuses de animaciones en texto o elementos de lectura. Respeta prefers-reduced-motion y evita bucles muy rápidos.

Definir keyframes

@keyframes fadeIn {
    from {
        opacity: 0;
    }
    to {
        opacity: 1;
    }
}

@keyframes bounce {
    0%, 100% {
        transform: translateY(0);
    }
    50% {
        transform: translateY(-20px);
    }
}

Aplicar animación

.element {
    animation-name: fadeIn;
    animation-duration: 0.5s;
    animation-timing-function: ease-out;
    animation-delay: 0s;
    animation-iteration-count: 1;     /* o infinite */
    animation-direction: normal;      /* reverse, alternate */
    animation-fill-mode: forwards;    /* none, backwards, both */
    animation-play-state: running;    /* paused */

    /* Atajo */
    animation: fadeIn 0.5s ease-out forwards;
    animation: bounce 1s ease-in-out infinite;
}

Definición de propiedades

  • @keyframes: define los pasos de la animación con porcentajes o from/to.
  • animation-name: nombre de los @keyframes a ejecutar.
  • animation-duration: tiempo total de la animación.
  • animation-timing-function: curva de velocidad dentro del ciclo.
  • animation-delay: espera antes de empezar.
  • animation-iteration-count: número de repeticiones o infinite.
  • animation-direction: dirección del ciclo (normal, reverse, alternate).
  • animation-fill-mode: define si se mantienen estilos antes/después (none, forwards, backwards, both).
  • animation-play-state: reproduce o pausa la animación.
  • animation (atajo): combina nombre, duración, función, delay, repeticiones, dirección y fill.

Accesibilidad: prefers-reduced-motion

@media (prefers-reduced-motion: reduce) {
    *, *::before, *::after {
        animation-duration: 0.01ms !important;
        animation-iteration-count: 1 !important;
        transition-duration: 0.01ms !important;
    }
}

Este media query reduce o elimina movimientos para usuarios sensibles al motion.

4Scroll-linked animations

2024 - Animaciones sincronizadas con el scroll.

Definición

Las scroll-linked animations sincronizan el progreso de una animación con el desplazamiento del usuario. Con scroll() el avance depende del scroll, y con view() depende de cómo entra el elemento en pantalla.

Cuándo usarla

Muy útiles para barras de progreso de lectura, revelar secciones al entrar, o hacer que cards de producto aparezcan suavemente al desplazarse en una tienda.

Recomendaciones

Ajusta animation-range para controlar el tramo visible y valida el soporte en navegador. Mantén los efectos sutiles para no distraer.

scroll() - Timeline anónima

@keyframes progress {
    from { width: 0%; }
    to { width: 100%; }
}

.progress-bar {
    animation: progress linear;
    animation-timeline: scroll();
    /* Se anima con el scroll de la página */
}

/* Especificar scroller */
.element {
    animation-timeline: scroll(root);   /* Viewport */
    animation-timeline: scroll(nearest); /* Contenedor más cercano */
}

view() - Entrada en viewport

@keyframes reveal {
    from {
        opacity: 0;
        transform: translateY(50px);
    }
    to {
        opacity: 1;
        transform: translateY(0);
    }
}

.fade-in {
    animation: reveal linear both;
    animation-timeline: view();
    animation-range: entry 0% cover 40%;
}

Definición de propiedades

  • animation-timeline: vincula la animación a una línea de tiempo de scroll o view.
  • scroll(): crea una timeline basada en el scroll (root o contenedor cercano).
  • view(): crea una timeline basada en la entrada/salida de un elemento en el viewport.
  • animation-range: define el tramo del scroll/viewport donde la animación progresa.
  • scroll-timeline-name: asigna un nombre reutilizable a una timeline de scroll.
  • scroll-timeline-axis: el eje que controla la timeline (block o inline).

scroll-timeline (con nombre)

.scroller {
    scroll-timeline-name: --my-timeline;
    scroll-timeline-axis: block;
}

.animated {
    animation: progress linear;
    animation-timeline: --my-timeline;
}

5View Transitions API

2023 - Transiciones entre páginas o estados de la aplicación.

Definición

View Transitions anima el cambio entre vistas: puedes “emparejar” un elemento en la vista vieja y la nueva para que parezca que se transforma en lugar de desaparecer.

Cuándo usarla

Ideal para transiciones entre páginas de un mismo sitio: portada → producto, lista → detalle, o cambios de sección en un panel. Da continuidad visual.

Recomendaciones

Empareja solo elementos clave (hero, títulos, imagen principal) para no saturar. Usa duraciones cortas y verifica soporte.

Transiciones entre páginas

/* Habilitar en CSS */
@view-transition {
    navigation: auto;
}

/* Nombrar elementos para transición */
.hero-image {
    view-transition-name: hero;
}

.page-title {
    view-transition-name: title;
}

Personalizar animaciones

/* Animación de salida */
::view-transition-old(hero) {
    animation: fade-out 0.3s ease-out;
}

/* Animación de entrada */
::view-transition-new(hero) {
    animation: fade-in 0.3s ease-out;
}

/* Para toda la página */
::view-transition-old(root) {
    animation: slide-out 0.3s;
}
::view-transition-new(root) {
    animation: slide-in 0.3s;
}

Definición de propiedades

  • @view-transition: habilita transiciones de navegación (por ejemplo, navigation: auto).
  • view-transition-name: vincula un elemento a una transición para emparejarlo entre vistas.
  • ::view-transition-old/new(...): pseudo-elementos para animar el estado de salida y entrada.
  • ::view-transition-old/new(root): permite animar toda la página.

@starting-style (animación de entrada)

Define el estado inicial para animar elementos que aparecen (dialog, popover).

dialog[open] {
    opacity: 1;
    transform: translateY(0);
}

dialog {
    opacity: 0;
    transform: translateY(-20px);
    transition: opacity 0.3s, transform 0.3s, display 0.3s allow-discrete;
}

@starting-style {
    dialog[open] {
        opacity: 0;
        transform: translateY(-20px);
    }
}

Definición de propiedades

  • @starting-style: define el estado inicial para transiciones de entrada.
  • display ... allow-discrete: permite transicionar cambios discretos (como display).
