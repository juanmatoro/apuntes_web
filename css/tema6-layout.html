0Estructura Semántica

Antes de aplicar Flexbox o Grid, debemos estructurar el contenido con etiquetas que aporten significado al navegador y buscadores.

Etiquetas de Bloque Principales

  • <header> Cabecera del sitio o de una sección (logo, nav).
  • <nav> Contenedor de enlaces de navegación principal.
  • <main> Contenido único y principal del documento.
  • <aside> Contenido tangencial (barras laterales, publicidad).
  • <footer> Pie de página (copyright, contacto, redes).

Esquema de Layout Típico 

<body>
  <header>...</header>
  <nav>...</nav>
  
  <main>
    <article>
      <section>...</section>
    </article>
    <aside>...</aside>
  </main>

  <footer>...</footer>
</body>

* Nota: El uso de estas etiquetas mejora el SEO y la accesibilidad (lectores de pantalla).

0.1Estructura y Altura Automática

Para que el layout ocupe siempre el 100% de la altura de la ventana (evitando que el footer "flote" a mitad de pantalla), aplicamos un modelo de Flexbox vertical al cuerpo del documento.

Sticky Footer con CSS

  • Min-height: 100vh/dvh: Asegura que el body sea al menos tan alto como la pantalla.
  • Flex-direction: column: Apila las secciones semánticas verticalmente.
  • Flex-grow: 1: Aplicado al main, obliga a esta sección a "estirarse" para rellenar todo el espacio sobrante, empujando el footer al final.
Tip: Usar min-height en lugar de height permite que la página crezca si el contenido supera el tamaño de la pantalla.

CSS Necesario

/* 1. Preparamos el contenedor */
body {
  display: flex;
  flex-direction: column;
  min-height: 100dvh; /* Altura dinámica */
  margin: 0;
}

/* 2. El contenido principal crece */
main {
  flex-grow: 1;
}

/* 3. Header y Footer mantienen su tamaño */
header, footer {
  flex-shrink: 0;
}

0.2El mismo Layout con Grid

El mismo resultado (footer siempre al fondo) se puede lograr con CSS Grid, usando filas explícitas y la unidad fr para que el contenido principal ocupe todo el espacio disponible.

Sticky Footer con Grid

  • Display: grid: Convierte el body en un contenedor de cuadrícula.
  • grid-template-rows: Definimos tres filas: auto para header y footer (se ajustan a su contenido) y 1fr para el main (ocupa todo el espacio restante).
  • Sin flex-grow: No hace falta asignar propiedades a los hijos; la definición de filas del contenedor es suficiente.
Tip: Con Grid defines la estructura completa en el contenedor, mientras que con Flexbox necesitas configurar también los hijos (flex-grow).

CSS Necesario

/* 1. Contenedor grid con 3 filas */
body {
  display: grid;
  grid-template-rows: auto 1fr auto;
  min-height: 100dvh;
  margin: 0;
}

/* 2. No necesitamos nada más.
   - header → fila auto (su contenido)
   - main   → fila 1fr  (todo el resto)
   - footer → fila auto (su contenido)
*/

Comparación rápida

Flex Configuras padre + hijos (flex-grow: 1)
Grid Solo configuras el padre (grid-template-rows)

1La propiedad display

Valores básicos

  • block - Ocupa todo el ancho, nueva línea
  • inline - Fluye con el texto, sin dimensiones
  • inline-block - Inline con dimensiones
  • none - Oculta completamente

Valores de layout

  • flex - Contenedor Flexbox
  • inline-flex - Flexbox inline
  • grid - Contenedor Grid
  • inline-grid - Grid inline

2Flexbox

Sistema de layout unidimensional (fila O columna).

Contenedor (padre) 

.container {
    display: flex;

    /* Dirección */
    flex-direction: row;        /* default */
    flex-direction: column;
    flex-direction: row-reverse;

    /* Wrap */
    flex-wrap: nowrap;          /* default */
    flex-wrap: wrap;

    /* Atajo */
    flex-flow: row wrap;

    /* Alineación eje principal */
    justify-content: flex-start;
    justify-content: center;
    justify-content: flex-end;
    justify-content: space-between;
    justify-content: space-around;
    justify-content: space-evenly;

    /* Alineación eje cruzado */
    align-items: stretch;       /* default */
    align-items: flex-start;
    align-items: center;
    align-items: flex-end;
    align-items: baseline;

    /* Múltiples líneas */
    align-content: flex-start;
}

Ítems (hijos) 

.item {
    /* Orden visual */
    order: 0;                   /* default */

    /* Crecimiento */
    flex-grow: 0;               /* No crece */
    flex-grow: 1;               /* Crece proporcionalmente */

    /* Encogimiento */
    flex-shrink: 1;             /* default, puede encogerse */

    /* Base del tamaño */
    flex-basis: auto;           /* Según contenido */
    flex-basis: 200px;          /* Tamaño inicial */

    /* Atajo */
    flex: 1;                    /* grow:1 shrink:1 basis:0 */
    flex: 0 0 200px;            /* No crece, no encoge, 200px */

    /* Alineación individual */
    align-self: center;
}

Centrar elemento 

.center {
    display: flex;
    justify-content: center;
    align-items: center;
}

2.1Flexbox: Guía de Propiedades

Sistema de distribución de espacio basado en un eje principal (main axis) y un eje cruzado (cross axis).

Container

  • flex-direction

    Define el eje principal: row (horizontal), column (vertical), o sus versiones reverse.

  • flex-wrap

    Controla si los ítems saltan a una nueva línea: nowrap (fuerza una línea), wrap (permite múltiples líneas).

  • justify-content

    Alineación en el eje principal. Distribuye el espacio sobrante entre o alrededor de los ítems.

  • align-items

    Alineación en el eje cruzado para los ítems dentro de una sola línea.

  • align-content

    Alineación de las líneas del grid (solo funciona si hay flex-wrap: wrap).

Items

  • flex-grow

    Capacidad de un ítem para crecer si hay espacio. 0 (fijo), 1 (ocupa lo disponible).

  • flex-shrink

    Capacidad de encogerse si no hay espacio. Por defecto es 1.

  • flex-basis

    Tamaño inicial del ítem antes de distribuir el espacio restante (reemplaza al width/height según el eje).

  • flex (Shorthand)

    Combinación de grow | shrink | basis. Ej: flex: 1 0 200px;

  • align-self

    Permite sobreescribir el align-items del padre para un ítem específico.

  • order

    Cambia el orden visual sin alterar el HTML. Por defecto 0.

Implementación Común 

.flex-parent {
  display: flex;
  flex-flow: row wrap; /* shorthand: direction + wrap */
  justify-content: space-between;
  align-items: center;
  gap: 1rem;
}

.flex-child {
  flex: 1 1 300px; /* Crece, encoge y base de 300px */
  align-self: stretch; /* Se estira verticalmente */
}

Recurso esencial: Para una guía visual completa de todas las propiedades, consulta A Complete Guide to Flexbox de CSS-Tricks.

3CSS Grid Layout

Sistema de maquetación bidimensional que permite trabajar con filas y columnas simultáneamente.

Definición del Contenedor

Se activa con display: grid. Define la estructura base de la cuadrícula.

.container {
    display: grid;
    /* 3 columnas: fija, flexible y flexible */
    grid-template-columns: 200px 1fr 2fr; 
    /* Filas automáticas */
    grid-template-rows: auto;
    /* Espacio entre celdas */
    gap: 20px; 
}

Posicionamiento de Ítems

Los hijos directos se pueden ubicar indicando líneas de inicio y fin.

.item-especial {
    /* Empieza en línea 1, ocupa 2 columnas */
    grid-column: 1 / 3; 
    /* También se puede usar span */
    grid-row: span 2; 
}

3.1Funciones y Áreas

Herramientas potentes para crear layouts dinámicos y legibles.

Funciones Mágicas

/* repeat() y minmax() */
grid-template-columns: repeat(3, 1fr);

/* Responsive sin Media Queries */
grid-template-columns: repeat(
    auto-fit, 
    minmax(200px, 1fr)
);

Grid Template Areas

.container {
    grid-template-areas:
        "header  header"
        "sidebar main"
        "footer  footer";
}
.header { grid-area: header; }

Recurso esencial: Domina la cuadrícula con la guía definitiva: A Complete Guide to CSS Grid de CSS-Tricks.

3.2Alineación de Contenidos

Grid permite controlar la posición de los elementos tanto a nivel de contenedor (tracks) como de celda individual (ítems).

Alineación del Contenedor (Tracks)

Distribuye el espacio sobrante si el grid es más pequeño que su contenedor.

.container {
    display: grid;
    /* Eje Horizontal (X) */
    justify-content: start | center | end | space-between;
    
    /* Eje Vertical (Y) */
    align-content: start | center | end | space-around;

    /* Atajo: place-content: [align] [justify] */
    place-content: center space-between;
}

Alineación de Ítems (Celdas)

Controla la posición del contenido dentro de su propia área asignada.

.container {
    /* Alineación global para todos los hijos */
    justify-items: start | center | end | stretch;
    align-items: start | center | end | stretch;
}

.item-especifico {
    /* Sobrescribir solo para este ítem */
    justify-self: center;
    align-self: end;
}

Eje Inline (Justify)

Alineación a lo largo de la fila (horizontal en español).

Eje Block (Align)

Alineación a lo largo de la columna (vertical en español).

Tip: Usa la unidad fr (fractional unit) para ocupar una fracción del espacio libre disponible, lo que facilita el diseño fluido sin cálculos manuales.

4Subgrid

Permite que un elemento hijo herede las líneas del grid padre. 

.grid {
    display: grid;
    grid-template-columns: repeat(4, 1fr);
    grid-template-rows: auto auto auto;
}

.card {
    grid-column: span 2;
    grid-row: span 3;

    /* Hereda las líneas del padre */
    display: grid;
    grid-template-columns: subgrid;
    grid-template-rows: subgrid;
}

/* Los hijos de .card se alinean
   con las líneas del grid principal */
.card-title { grid-row: 1; }
.card-body { grid-row: 2; }
.card-footer { grid-row: 3; }

Uso: Alinear elementos de tarjetas (título, contenido, footer) entre sí, incluso con contenido de diferentes longitudes.

5Gap

2021 - Espaciado entre elementos en Flex y Grid. 

.container {
    display: flex; /* o grid */

    /* Gap uniforme */
    gap: 20px;

    /* Filas y columnas diferentes */
    gap: 10px 20px;      /* row-gap | column-gap */

    /* Propiedades individuales */
    row-gap: 10px;
    column-gap: 20px;
}

Ventajas sobre margin

  • Solo aplica entre elementos (no en bordes)
  • No necesita selectores :first-child/:last-child
  • No hay colapso de márgenes
  • Funciona con wrap en Flexbox

6Multi-column layout

Divide el contenido en múltiples columnas, como en periódicos.

Propiedades principales 

.article {
    /* Número de columnas */
    column-count: 3;

    /* O ancho de columna (auto-calcula cantidad) */
    column-width: 200px;

    /* Atajo: auto para uno, valor para otro */
    columns: 3;
    columns: 200px;
    columns: 3 200px;

    /* Espacio entre columnas */
    column-gap: 40px;

    /* Línea separadora */
    column-rule: 1px solid #ccc;
}

Control de contenido 

/* Evitar que un elemento se corte entre columnas */
.article h2 {
    break-after: avoid;
}

.article figure {
    break-inside: avoid;
}

/* Elemento que ocupa todas las columnas */
.article .featured {
    column-span: all;
}

Uso: Artículos largos, galerías tipo masonry, listas en columnas.

7Padding vs Margin: ¿Cuál usar?

Aunque ambos crean "espacio en blanco", su comportamiento técnico es radicalmente opuesto.

Diferencias Clave

Padding (Interno)

Es el espacio dentro del borde. Afecta al tamaño del elemento y muestra el color de fondo.

Margin (Externo)

Es el espacio fuera del borde. Separa al elemento de sus vecinos y es siempre transparente.

  • Clickability: El padding aumenta el área clickable (en botones); el margin no.
  • Colapso: Los márgenes verticales pueden "colapsar" (unirse), el padding nunca.
  • Fondo: El `background-color` se ve en el padding, pero no en el margin.
  • Regla de oro de componentes: Un componente bien diseñado debería definir su propio Padding (cómo se ve por dentro), pero lo ideal es que el Margin (cómo se separa de otros) lo defina el contenedor padre o una clase de utilidad, para que el componente sea reutilizable en cualquier sitio.

Reglas de Oro

/* ✅ BIEN: Separación externa */
.card {
    margin-bottom: 20px; 
}

/* ✅ BIEN: Respiración interna */
.card-content {
    padding: 1.5rem;
    background-color: white;
}

/* ❌ MAL: Usar margin para separar el 
   texto del borde de su propio contenedor */
.bad-card p {
    margin: 20px; /* El fondo se cortará antes del texto */
}

Consejo Pro: Si el elemento tiene un borde o un fondo, usa padding para que el contenido no lo toque. Si quieres alejarte de otros elementos, usa margin.
