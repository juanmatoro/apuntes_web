1Colores básicos

Hexadecimal

.color {
    color: #ff0000;      /* Rojo */
    color: #f00;         /* Abreviado */
    color: #ff000080;    /* Con alpha (50%) */
}

RGB / RGBA

.color {
    color: rgb(255, 0, 0);
    color: rgb(255 0 0);        /* Sintaxis moderna */
    color: rgb(255 0 0 / 50%);  /* Con alpha */
    color: rgba(255, 0, 0, 0.5);
}

HSL / HSLA

Hue (0-360), Saturation (%), Lightness (%)

.color {
    color: hsl(0, 100%, 50%);     /* Rojo */
    color: hsl(0 100% 50%);       /* Sintaxis moderna */
    color: hsl(0 100% 50% / 50%); /* Con alpha */
}

Palabras clave

  • Colores nombrados: red, blue, tomato, etc.
  • transparent, currentColor

2Colores de gama amplia

2023 - Espacios de color que van más allá de sRGB.

HWB

Hue, Whiteness, Blackness - Más intuitivo que HSL

.color {
    color: hwb(0 0% 0%);    /* Rojo puro */
    color: hwb(0 50% 0%);   /* Rojo claro */
    color: hwb(0 0% 50%);   /* Rojo oscuro */
}

LAB / LCH

Basados en percepción humana. Mayor gama de colores.

.color {
    color: lab(50% 40 60);       /* L, a, b */
    color: lch(50% 70 30);       /* L, Chroma, Hue */
}

OKLAB / OKLCH

Versión mejorada de LAB/LCH. Mejor para gradientes.

.color {
    color: oklab(0.6 0.1 0.1);
    color: oklch(0.6 0.15 30);   /* Recomendado */
}

/* Gradiente sin colores "sucios" */
.gradient {
    background: linear-gradient(
        in oklch,
        red, blue
    );
}

color() - Espacio personalizado

.vivid-green {
    background: color(display-p3 0 1 0);
}

3Herramientas de color

color-mix() (2022)

Mezcla dos colores en cualquier espacio.

.mixed {
    /* 50% de cada color por defecto */
    background: color-mix(in srgb, red, blue);

    /* Proporciones personalizadas */
    background: color-mix(in oklch, red 30%, blue);

    /* Crear variantes */
    --light: color-mix(in oklch, var(--brand) 80%, white);
    --dark: color-mix(in oklch, var(--brand) 80%, black);
}

Sintaxis relativa (2024)

Modifica un color existente con la palabra clave from.

.variants {
    --base: #4488dd;

    /* Aclarar */
    --light: hsl(from var(--base) h s calc(l * 1.3));

    /* Oscurecer */
    --dark: hsl(from var(--base) h s calc(l * 0.7));

    /* Cambiar saturación */
    --muted: hsl(from var(--base) h calc(s * 0.5) l);

    /* Complementario */
    --complement: hsl(from var(--base) calc(h + 180) s l);
}

accent-color (2021)

Color de acento para controles del navegador.

input[type="checkbox"],
input[type="radio"],
input[type="range"],
progress {
    accent-color: #3b82f6;
}

Dónde usar color

  • color - Texto
  • background-color
  • border-color
  • outline-color
  • box-shadow, text-shadow
  • fill, stroke (SVG)

4Unidades de medida

Absolutas

Mantienen su tamaño independientemente de la configuración de la pantalla.

  • px - **Píxeles:** La unidad base digital (1px = 1 punto en pantalla).
  • cm, mm, in, pt - **Físicas:** Ideales para @media print.

Relativas al texto

Se ajustan según el tamaño de la fuente, mejorando la accesibilidad.

  • rem - **Root EM:** Relativo al font-size del `` (por defecto 16px). **Es la más recomendada.**
  • em - **Elemento:** Relativo al tamaño de fuente del elemento padre (o del propio elemento).
  • ch - **Character:** Ancho del carácter "0". Perfecto para limitar el ancho de párrafos y mejorar lectura.
  • ex - **X-height:** Altura de la letra "x" minúscula de la fuente actual.
        /* Si el root es 16px... */
        h1 { font-size: 2.5rem; }      /* 16 * 2.5 = 40px (Siempre fijo) */

        .card { font-size: 20px; }
        .card p { font-size: 1.5em; }  /* 20 * 1.5 = 30px (Relativo al padre) */

        .txt { max-width: 60ch; }      /* El ancho ideal para lectura cómoda */

Porcentajes y Viewport

Dimensiones basadas en el contenedor o el tamaño de la ventana.

  • % - Relativo al tamaño del contenedor padre directo.
  • vw / vh - 1% del ancho / alto de la ventana del navegador (Viewport).
    .container { width: 80%; }     /* 80% del ancho del padre */
    .hero { height: 100vh; }       /* Ocupa toda la altura de la pantalla */
    .half { width: 50vw; }         /* Ocupa la mitad del ancho de pantalla */

5Unidades de viewport

Clásicas

Basadas en el tamaño total de la ventana (incluyendo barras de UI).

  • vw - **1% Viewport Width:** Relativo al ancho total de la pantalla.
  • vh - **1% Viewport Height:** Relativo al alto total de la pantalla.
  • vmin - Toma el valor **más pequeño** entre vw y vh (útil para que algo quepa siempre en pantalla).
  • vmax - Toma el valor **más grande** entre vw y vh.

Variables Modernas (Solución para Móviles)

Ajustan el alto ignorando o considerando las barras del navegador.

  • svh / svw - **Small:** El espacio disponible cuando las barras del navegador están **expandidas** (el área más pequeña).
  • lvh / lvw - **Large:** El espacio cuando las barras están **ocultas** (el área más grande).
  • dvh / dvw - **Dynamic:** Se ajusta en tiempo real según si el usuario hace scroll y las barras aparecen o desaparecen.
.full-screen-hero {
    min-height: 100dvh; /* Evita que el botón 'hacer scroll' quede tapado */
}

.modal {
    max-height: 90svh;  /* Asegura que el modal no se corte con el navegador */
}

Unidades Lógicas

Dependen de la dirección del texto (idiomas horizontales vs verticales).

  • vi - **Inline:** Equivale al ancho en español, pero al alto en japonés vertical.
  • vb - **Block:** Equivale al alto en español, pero al ancho en japonés vertical.

6Propiedades lógicas

Reemplazan left/right/top/bottom por propiedades que se adaptan al modo de escritura (LTR, RTL, vertical).

Flujo inline (horizontal)

  • margin-inline = margin-left + margin-right
  • margin-inline-start = margin-left (en LTR)
  • margin-inline-end = margin-right (en LTR)
  • Igual para padding-inline, border-inline

Flujo block (vertical)

  • margin-block = margin-top + margin-bottom
  • margin-block-start = margin-top
  • margin-block-end = margin-bottom
  • Igual para padding-block, border-block

Tamaño

  • inline-size = width
  • block-size = height
  • max-inline-size = max-width

Posicionamiento (inset)

  • inset = top + right + bottom + left
  • inset-inline = left + right
  • inset-block = top + bottom
/* Antes (solo LTR) */
.card {
    margin-left: auto;
    margin-right: auto;
    padding-top: 2rem;
    padding-bottom: 2rem;
}

/* Ahora (funciona en LTR y RTL) */
.card {
    margin-inline: auto;
    padding-block: 2rem;
}

/* Alineación de texto */
.text {
    text-align: start;  /* left en LTR, right en RTL */
    text-align: end;    /* right en LTR, left en RTL */
}
