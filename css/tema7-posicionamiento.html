1Position

static (default)

Flujo normal del documento. top/left/etc no funcionan.

.element {
    position: static;
    /* top, left, etc. no tienen efecto */
}

relative

Se desplaza respecto a su posición original. Mantiene su espacio.

.element {
    position: relative;
    top: 10px;    /* Baja 10px */
    left: 20px;   /* Se mueve 20px a la derecha */
}

absolute

Sale del flujo. Se posiciona respecto al ancestro posicionado más cercano.

.parent {
    position: relative;  /* Referencia */
}
.child {
    position: absolute;
    top: 0;
    right: 0;           /* Esquina superior derecha del padre */
}

fixed

Sale del flujo. Se posiciona respecto al viewport.

.navbar {
    position: fixed;
    top: 0;
    left: 0;
    right: 0;
    /* Siempre visible arriba */
}

.back-to-top {
    position: fixed;
    bottom: 20px;
    right: 20px;
}

2Position sticky

2020 - Híbrido entre relative y fixed.

Se comporta como relative hasta que alcanza un umbral, luego se "pega".

.sticky-header {
    position: sticky;
    top: 0;              /* Se pega cuando llega arriba */
    background: white;
    z-index: 100;
}

.sidebar {
    position: sticky;
    top: 80px;           /* Se pega a 80px del top */
}

Requisitos

  • Necesita al menos un valor de top/bottom/left/right
  • El contenedor padre debe tener altura suficiente
  • No funciona si un ancestro tiene overflow: hidden

3Anchor positioning

2023 - Posiciona elementos relativos a otros elementos "ancla". 

/* Definir el ancla */
.trigger {
    anchor-name: --my-anchor;
}

/* Posicionar relativo al ancla */
.tooltip {
    position: absolute;
    position-anchor: --my-anchor;

    /* Posición relativa al ancla */
    top: anchor(bottom);
    left: anchor(center);

    /* O usando inset-area */
    inset-area: bottom center;
}

Uso: Tooltips, dropdowns, popovers que se posicionan automáticamente respecto a su trigger.

4Z-index y capas

Controla el orden de apilamiento (qué elemento aparece encima).

.element {
    position: relative;  /* Necesario para z-index */
    z-index: 10;
}

/* Sistema de capas recomendado */
:root {
    --z-dropdown: 100;
    --z-sticky: 200;
    --z-fixed: 300;
    --z-modal-backdrop: 400;
    --z-modal: 500;
    --z-tooltip: 600;
}

Contexto de apilamiento

  • Solo funciona en elementos posicionados (no static)
  • También en flex/grid items y con opacity < 1
  • z-index negativo coloca detrás del padre
  • Cada contexto es independiente

5Inset (propiedad lógica)

Atajo para top, right, bottom, left.

/* Equivalentes */
.overlay {
    position: absolute;
    top: 0; right: 0; bottom: 0; left: 0;

    /* Es igual a: */
    inset: 0;
}

/* Valores individuales */
.element {
    inset: 10px;              /* Todos */
    inset: 10px 20px;         /* Vertical | Horizontal */
    inset: 10px 20px 30px;    /* Top | Horizontal | Bottom */
    inset: 10px 20px 30px 40px; /* T | R | B | L */
}

/* Propiedades lógicas */
.element {
    inset-block: 10px;        /* top + bottom */
    inset-inline: 20px;       /* left + right */
    inset-block-start: 10px;  /* top */
}

6Márgenes y espaciado

Colapso de márgenes

Los márgenes verticales adyacentes se colapsan (el mayor gana).

/* Si ambos tienen margin, solo se aplica el mayor */
.a { margin-bottom: 20px; }
.b { margin-top: 30px; }
/* Espacio entre ellos: 30px (no 50px) */

/* Evitar colapso */
.parent {
    display: flex;
    flex-direction: column;
    /* o */
    display: flow-root;
}

Margin auto

/* Centrar horizontalmente */
.centered {
    width: 80%;
    margin-inline: auto;
}

/* Empujar en flexbox */
.spacer {
    margin-left: auto;  /* Empuja todo a la derecha */
}

Margin negativo

.pull-up {
    margin-top: -20px;  /* Sube y superpone */
}

/* Full-bleed en contenedor con padding */
.full-width {
    margin-inline: -20px;
    padding-inline: 20px;
}
