1Pseudo-elementos (::)

Permiten estilizar partes específicas de un elemento. Se escriben con doble dos puntos (::).

::before y ::after

Insertan contenido antes/después del contenido del elemento. Requieren content.

.quote::before {
    content: "\201C"; /* Comilla " */
    font-size: 2em;
    color: gray;
}
.external-link::after {
    content: " ↗";
}

::first-letter y ::first-line

Estilizan la primera letra o línea de un bloque de texto.

p::first-letter {
    font-size: 3em;
    float: left;
    line-height: 1;
}
p::first-line {
    font-weight: bold;
}

::marker

Estiliza el marcador de listas (viñetas o números).

li::marker {
    color: tomato;
    font-weight: bold;
}
summary::marker {
    content: "▶ ";
}

::selection

Estiliza el texto seleccionado por el usuario.

::selection {
    background: #ffb7b7;
    color: #8b0000;
}

::placeholder

Estiliza el texto placeholder de inputs.

input::placeholder {
    color: #999;
    font-style: italic;
}

::backdrop

Estiliza el fondo detrás de elementos en pantalla completa (dialog, video).

dialog::backdrop {
    background: rgba(0,0,0,0.7);
    backdrop-filter: blur(4px);
}

2Estados interactivos

Pseudo-clases que responden a la interacción del usuario.

  • :hover - Cursor sobre el elemento
  • :active - Elemento siendo activado (clic)
  • :focus - Elemento con foco (teclado/clic)
  • :focus-visible - Foco solo cuando es necesario mostrarlo (2022)
  • :focus-within - Padre contiene elemento con foco
  • :target - Elemento apuntado por ancla URL (#id)
.btn:hover { background: #0056b3; }
.btn:active { transform: scale(0.98); }

/* Solo mostrar outline con teclado */
.btn:focus-visible {
    outline: 2px dashed orange;
}

/* Resaltar formulario cuando un input tiene foco */
form:focus-within {
    box-shadow: 0 0 0 3px rgba(0,123,255,0.25);
}

Estados históricos (enlaces)

  • :link - Enlace no visitado
  • :visited - Enlace ya visitado

3Estados de formulario

Para estilizar inputs según su estado.

Estado general

  • :enabled / :disabled
  • :read-only / :read-write
  • :placeholder-shown - Muestra placeholder

Estado de selección

  • :checked - Checkbox/radio marcado
  • :indeterminate - Estado intermedio

Validación

  • :valid / :invalid
  • :required / :optional
  • :in-range / :out-of-range
input:disabled { opacity: 0.5; }
input:valid { border-color: green; }
input:invalid { border-color: red; }
input:required { border-left: 3px solid orange; }

4Selección de elementos

Para seleccionar elementos por su posición.

Por posición (hijo)

  • :first-child / :last-child
  • :nth-child(n) - Enésimo hijo
  • :nth-last-child(n) - Desde el final
  • :only-child - Hijo único

Por tipo

  • :first-of-type / :last-of-type
  • :nth-of-type(n)
  • :only-of-type
/* Fórmulas nth-child */
:nth-child(odd)      /* Impares: 1, 3, 5... */
:nth-child(even)     /* Pares: 2, 4, 6... */
:nth-child(3n)       /* Múltiplos de 3 */
:nth-child(3n+1)     /* 1, 4, 7, 10... */
:nth-child(-n+3)     /* Primeros 3 */

/* 2023: nth-child con selector */
:nth-child(2 of .active) /* 2do elemento .active */

5Búsqueda y coincidencia

:not() - Negación

Selecciona elementos que NO coinciden.

p:not(.intro) { }      /* p sin clase .intro */
input:not([disabled]) { }  /* inputs habilitados */

:is() - Coincidencia

Agrupa selectores sin repetir. Toma la especificidad más alta.

:is(h1, h2, h3) { color: navy; }
article :is(h1, h2, h3) { margin-top: 2rem; }

:where() - Sin especificidad (2022)

Igual que :is() pero con especificidad 0. Ideal para resets.

:where(h1, h2, h3) { margin: 0; } /* Fácil de sobrescribir */

:has() - Selector padre (2022)

Revolucionario: permite seleccionar un elemento basado en sus hijos/hermanos. 

/* Card que contiene imagen */
.card:has(img) { padding: 0; }

/* Formulario con input inválido */
form:has(:invalid) { border-color: red; }

/* Label seguido de input requerido */
label:has(+ input:required)::after {
    content: " *";
    color: red;
}

:empty

Elementos sin contenido (ni texto ni hijos).

p:empty { display: none; }
