1Objetos básicos

Un objeto es un dato compuesto de tipo object formado por pares clave-valor que representan una entidad o concepto. Cada clave (nombre de propiedad) es string o symbol, y cada valor puede ser cualquier tipo de dato.

  • Ideal para agrupar datos relacionados bajo un mismo nombre.
  • El orden de las propiedades no es lo importante, lo importante es la clave.
  • Modelo mental: piensa en una "ficha" con etiquetas (claves) y su contenido (valores).

Modelo mental del objeto

const persona = {
  nombre: 'Ana',    // clave: nombre  | valor: 'Ana' (string)
  edad: 25,         // clave: edad    | valor: 25 (number)
  activo: true      // clave: activo  | valor: true (boolean)
};

// Tipo de dato del objeto completo:
typeof persona; // "object"

Literal

const user = {
  nombre: 'Ana',
  edad: 25,
  activo: true
};

Acceso

user.nombre;      // 'Ana'
user['edad'];     // 25
const key = 'activo';
user[key];        // true

2Propiedades y métodos

Las propiedades almacenan datos y los métodos encapsulan comportamiento. Un método es una función asociada a un objeto que suele usar this.

  • Usa this para acceder a otras propiedades del mismo objeto.
  • Las claves dinámicas permiten crear propiedades con valores calculados.

Métodos

const user = {
  nombre: 'Ana',
  saludar() {
    return `Hola ${this.nombre}`;
  }
};

Claves dinámicas

const field = 'email';
const user = {
  [field]: 'ana@mail.com'
};

3Referencia y mutabilidad

Los objetos se comparan por referencia. Dos objetos con el mismo contenido no son iguales si son referencias distintas.

  • Asignar un objeto a otra variable copia la referencia, no el contenido.
  • El spread { ...obj } crea una copia superficial.

Referencia

const a = { x: 1 };
const b = a;

b.x = 2;
a.x; // 2

Copia superficial

const a = { x: 1, y: 2 };
const b = { ...a };

b.y = 9;
a.y; // 2

4Destructuring y rest

Extrae propiedades en variables de forma concisa. Puedes renombrar, dar valores por defecto y recoger el resto.

  • Mejora la legibilidad cuando solo necesitas algunas propiedades.
  • El operador rest agrupa las propiedades restantes en un nuevo objeto.

Destructuring

const user = { nombre: 'Ana', edad: 25 };
const { nombre, edad } = user;

Rest

const { nombre, ...resto } = user;
resto; // { edad: 25 }

5Patrones de uso

Los objetos son ideales para representar entidades y pasar configuración. Ayudan a evitar listas largas de parámetros.

  • Modela entidades con propiedades consistentes y predecibles.
  • Pasa opciones como un objeto para extensibilidad.

Entidad

const task = {
  id: 1,
  title: 'Leer',
  done: false
};

Configuración

function init({ host = 'localhost', port = 3000 }) {
  return `http://${host}:${port}`;
}
