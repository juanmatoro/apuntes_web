1Modelo de ejecución de JavaScript

JavaScript ejecuta código principal en un hilo de ejecución (main thread). El modelo se apoya en call stack, Web/Node APIs, colas de tareas y event loop.

  • Call stack: pila de funciones que se están ejecutando ahora.
  • Heap: memoria de objetos y estructuras dinámicas.
  • Task queue: callbacks de timers/eventos listos para ejecutar.
  • Microtask queue: promesas resueltas y queueMicrotask.
console.log('inicio');
setTimeout(() => console.log('timer'), 0);
Promise.resolve().then(() => console.log('microtask'));
console.log('fin');
// salida: inicio, fin, microtask, timer

2Colas y prioridades: macro vs micro

Tras cada bloque de código sincrónico, el event loop vacía primero microtareas y luego atiende tareas de cola principal. Esto afecta orden, fluidez y latencia percibida.

Tipo Ejemplos Prioridad
Microtask Promise.then, await, queueMicrotask Alta
Task (macro) setTimeout, eventos UI, I/O callbacks Normal

Si encadenas demasiadas microtareas puedes bloquear render y eventos de usuario aunque no uses bucles pesados.

3Patrones clásicos de ejecución

Secuencial

Ejecuta una tarea y espera su final antes de iniciar la siguiente. Es simple pero lento cuando no hay dependencia real.

const a = await getA();
const b = await getB();
const c = await getC();

Concurrente

Lanza varias tareas y espera resultados en conjunto. Reduce tiempo total de espera de I/O.

const [a, b, c] = await Promise.all([
  getA(), getB(), getC()
]);

Variantes de orquestación

  • Promise.all: falla rápido si una promesa falla.
  • Promise.allSettled: devuelve éxitos y fallos sin cortar flujo.
  • Promise.race: gana la primera (ok o error).
  • Promise.any: primera exitosa; ignora fallos hasta agotar todas.

4Control de concurrencia y backpressure

Lanzar cientos de promesas en paralelo puede saturar red, API o navegador. El patrón recomendado es limitar concurrencia por lotes o semáforos.

async function ejecutarEnLotes(items, size, worker) {
  const out = [];
  for (let i = 0; i < items.length; i += size) {
    const lote = items.slice(i, i + size);
    const parcial = await Promise.all(lote.map(worker));
    out.push(...parcial);
  }
  return out;
}

A esto se le llama backpressure: regular ritmo de producción para que el consumidor no colapse.

5Antipatrones frecuentes

  • Await dentro de bucle sin necesidad: vuelve secuencial una tarea paralelizable.
  • Promesas sin captura: generan errores no controlados (unhandledrejection).
  • Abusar de microtasks: puede retrasar paint y eventos de usuario.
  • No cancelar trabajo obsoleto: respuestas viejas pisan estado reciente.

Regla operativa: secuencial solo cuando hay dependencia; concurrente controlada para I/O independiente; cancelar y limitar cuando la carga crece.
