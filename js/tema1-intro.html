Arquitecturas web Navegadores Motores y Entornos JS

1Arquitecturas de aplicaciones web

Cliente / Servidor

La arquitectura web se basa en el modelo cliente-servidor: el navegador (cliente) solicita recursos al servidor, que los procesa y envía una respuesta.

Modelos de navegación

  • SPA (Single Page Application) — Una sola página HTML que se actualiza dinámicamente sin recargar el navegador
  • MPA (Multi Page Application) — Múltiples páginas HTML, cada navegación implica una nueva solicitud al servidor

Modelos de renderizado

Modelo Descripción Momento
Creación manual HTML generado manualmente en el servidor Servidor
SSG Static Site Generation — HTML generado en el servidor en tiempo de compilación Ahead of time
SSR Server Side Rendering — HTML generado en el servidor en tiempo de ejecución Just in time
CSR Client Side Rendering — HTML generado en el cliente en tiempo de ejecución Just in time

Otros términos

  • JAMStack — JavaScript, APIs y Markup: arquitectura basada en pre-renderizado y desacoplamiento
  • PWA (Progressive Web App) — Aplicaciones web con capacidades nativas (offline, instalable)
  • Microservicios — Arquitectura donde la aplicación se divide en servicios independientes
  • Serverless — Ejecución de funciones en la nube sin gestionar servidores
  • Headless CMS — Sistema de gestión de contenido que expone los datos vía API

2Navegadores web

Componentes de un navegador

  • Interfaz de usuario — Barra de direcciones, botones, marcadores...
  • Motor de navegación — Coordina las acciones entre el UI y el motor de renderizado
  • Motor de renderizado — Presenta el contenido visual del documento
  • Persistencia de datos — Almacenamiento local (cookies, localStorage, IndexedDB)
  • Networking — Gestiona las peticiones HTTP/HTTPS
  • Intérprete de scripts — Motor de JavaScript
  • Backend del UI — Renderiza elementos básicos (ventanas, combos, etc.)

Motor de renderizado

El rendering engine es el responsable de presentar el contenido del documento en pantalla. Su flujo de trabajo:

  1. Parsing — Análisis sintáctico del HTML y CSS, transformándolos en un árbol de derivación
  2. Árbol DOM — Creación del Document Object Model que representa la estructura del documento
  3. Reglas de estilo — Aplicación de las reglas CSS al árbol DOM
  4. Render Tree — Creación del árbol de renderizado con la información visual
  5. Layout — Disposición de los elementos en la pantalla
  6. Paint — Presentación final del contenido en la pantalla
HTML/CSS → Parsing → Árbol DOM + CSSOM → Render Tree → Layout → Paint

3Motores y Entornos de ejecución de JS

Motor de JavaScript

El intérprete de scripts (motor de JS) es el responsable de ejecutar el código JavaScript. Interpreta el código fuente, genera código intermedio (bytecode) y lo ejecuta.

Principales motores

Motor Usado en
V8 Chrome, Node.js, Edge (Chromium)
SpiderMonkey Firefox
JavaScriptCore Safari
Chakra Edge (Legacy)

Núcleo del lenguaje (core)

Todos los motores implementan el núcleo de JavaScript con una API mínima:

  • Manejo de datos, textos, arrays y expresiones regulares
  • No hay funcionalidad de entradas y salidas (I/O)
  • No hay funciones más sofisticadas: redes, almacenamiento, gráficos…

Entornos de ejecución

Los motores se integran en diversos entornos que proporcionan APIs específicas:

Navegadores

  • Integra motor JS + motor de renderizado + backend UI
  • API: DOM, Web APIs, fetch, localStorage…

Node.js

  • Entorno de ejecución fuera del navegador (motor V8)
  • API: fs, http, modules, process…

Transpiladores

Herramientas como Babel transforman código JavaScript moderno (ES6+) en versiones compatibles con navegadores antiguos. Permiten usar las últimas características del lenguaje sin preocuparse por la compatibilidad.

// Código fuente (ES6+)
const greet = (name) => `Hola, ${name}!`;

// Transpilado (ES5)
var greet = function(name) {
    return "Hola, " + name + "!";
};
