1Fundamentos

La programación funcional construye soluciones con funciones pequeñas, composables y predecibles. En JavaScript no es exclusiva: convive con estilos imperativo y orientado a objetos.

  • Declarativo: describes qué transformar, no cada paso interno.
  • Funciones como valores: se pasan, retornan y almacenan.
  • Composición: combinar piezas simples para resolver problemas complejos.

2Funciones puras y efectos colaterales

Una función pura siempre devuelve el mismo resultado para la misma entrada y no modifica estado externo.

// Pura
const aplicarIVA = (importe, tipo) => importe * (1 + tipo);

// Impura (depende de estado global)
let descuentoGlobal = 0.1;
const calcular = (precio) => precio * (1 - descuentoGlobal);

// Impura (efecto colateral)
const loguear = (msg) => console.log(msg);

3Inmutabilidad

Evita mutar arrays u objetos originales; crea nuevas estructuras con ....

const usuario = { id: 1, nombre: 'Ana', activo: true };

// mutación (evitar)
// usuario.activo = false;

// inmutable (recomendado)
const usuarioActualizado = { ...usuario, activo: false };

const nums = [1, 2, 3];
const conCuatro = [...nums, 4];

4Composición de funciones

La composición encadena transformaciones pequeñas. Es la base de utilidades reutilizables.

const trim = (s) => s.trim();
const lower = (s) => s.toLowerCase();
const slug = (s) => s.replace(/\s+/g, '-');

const toSlug = (texto) => slug(lower(trim(texto)));
toSlug('  Curso JavaScript Moderno  '); // 'curso-javascript-moderno'

5Pipeline con map, filter y reduce

Para procesar colecciones: filtras, transformas y acumulas en una secuencia legible.

const pedidos = [
  { id: 1, total: 120, pagado: true },
  { id: 2, total: 80, pagado: false },
  { id: 3, total: 240, pagado: true }
];

const facturacion = pedidos
  .filter((p) => p.pagado)
  .map((p) => p.total)
  .reduce((acc, total) => acc + total, 0);

// facturacion = 360
