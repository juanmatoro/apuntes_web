1Client-Side Rendering (CSR)

En CSR el servidor entrega un HTML prácticamente vacío. Es JavaScript quien genera y actualiza todo el contenido en el navegador, manipulando el DOM directamente. Este modelo es la base de la arquitectura de componentes.

SSR — Server-Side Rendering

  • El servidor genera el HTML completo.
  • El navegador recibe HTML ya renderizado.
  • Cada navegación recarga la página.
  • Menor control dinámico desde el cliente.

CSR — Client-Side Rendering

  • El servidor entrega un HTML mínimo (shell).
  • JS construye la UI en el navegador.
  • Las actualizaciones no recargan la página.
  • Permite componentes reutilizables y estado.

Flujo CSR

1. Navegador pide la página → 2. Servidor devuelve HTML vacío + <script> → 3. JS se ejecuta y renderiza componentes → 4. Eventos actualizan el estado → re-render

HTML shell típico en CSR

<!-- El servidor solo entrega esto -->
<body>
  <div id="app"></div>   <!-- contenedor vacío -->
  <script src="app.js"></script>  <!-- JS construye todo -->
</body>

Punto de entrada del render

const state = { items: [] };

function renderApp() {
  const root = document.getElementById('app');
  root.innerHTML = App(state);   // genera HTML desde JS
  bindEvents();                  // re-enlaza eventos tras cada render
}

renderApp();  // primer render al cargar la página
Clave: en CSR la fuente de verdad es el estado en memoria (un objeto JS), no el HTML. Cuando el estado cambia, se vuelve a generar el HTML completo del componente afectado.

1Modelo de componente

Un componente encapsula estructura, comportamiento y reglas de actualización de una parte de la UI.

function TodoItem(todo) {
  return `<li data-id="${todo.id}">${todo.texto}</li>`;
}

2Render y composición

Componer componentes es combinar piezas pequeñas para construir vistas completas.

function TodoList(todos) {
  const items = todos.map(TodoItem).join('');
  return `<ul>${items}</ul>`;
}

3Estado y actualización

El estado vive en una capa central y la vista se vuelve a renderizar cuando ese estado cambia.

const state = { todos: [] };

function setState(partial) {
  Object.assign(state, partial);
  renderApp();
}

4Flujo CRUD básico

Operaciones

  • Create: agregar elemento
  • Read: listar elementos
  • Update: editar estado
  • Delete: eliminar elemento

Ejemplo

function addTodo(texto) {
  const todo = { id: crypto.randomUUID(), texto };
  setState({ todos: [...state.todos, todo] });
}

5Recursos y herramientas

  • Composición por funciones puras para vistas simples.
  • Separar estado, render y handlers.
  • Escalar a framework cuando crece la complejidad.
