1Carga de scripts en HTML

Antes de importar archivos JS entre sí, conviene dominar cómo el navegador carga scripts.

Formas principales

<script src="app.js"></script>
<script defer src="app.js"></script>
<script async src="analytics.js"></script>
<script type="module" src="main.js"></script>

Cuándo usar cada una

  • defer: scripts de la app sin bloquear render.
  • async: scripts independientes (analytics, ads).
  • type="module": cuando usarás import/export.

2ES Modules: export e import

Un módulo es un archivo JS con su propio scope. Expone lo necesario con export y consume con import.

Archivo exportador

// math.js
export const PI = 3.1416;

export function sumar(a, b) {
  return a + b;
}

Archivo consumidor

// main.js
import { PI, sumar } from './math.js';

console.log(PI);
console.log(sumar(2, 3));

3Tipos y formas de importación

Exportaciones

// named export
export const API_URL = '/api';

// default export
export default function initApp() {
  console.log('init');
}

Importaciones

import initApp from './init.js';
import { API_URL } from './config.js';
import { API_URL as URL } from './config.js';
import * as utils from './utils.js';

4Importación dinámica (import())

Permite cargar módulos bajo demanda (lazy loading), por ejemplo cuando el usuario abre una vista pesada.

btnReportes.addEventListener('click', async () => {
  const { renderReportes } = await import('./reportes.js');
  renderReportes();
});

5ESM vs CommonJS

En navegador moderno el estándar es ESM. En Node puedes encontrar ambos formatos.

ESM

import { readFile } from 'node:fs/promises';
export const version = '1.0.0';

CommonJS

const fs = require('node:fs');
module.exports = { version: '1.0.0' };

6Buenas prácticas de módulos

  • Un módulo por responsabilidad (UI, servicios, utilidades, estado).
  • Usar rutas relativas claras y extensión .js en navegador.
  • Evitar dependencias circulares entre archivos.
  • No abusar de export default en todo; combina con exports nombrados.
