1Definición y partes de una función

Una función es un bloque de código reutilizable que puede recibir datos de entrada, ejecutar una tarea y devolver un resultado.

Parte Qué hace Ejemplo
Nombre Identifica la función para poder invocarla. saludar
Parámetros Variables de entrada que recibe la función. (nombre)
Cuerpo Instrucciones que se ejecutan al llamar la función. { ... }
Valor de retorno Resultado que entrega con return. return mensaje;
Invocación Ejecución de la función. saludar('Ana') 
function saludar(nombre) {          // nombre + parámetro
  const mensaje = `Hola ${nombre}`;  // cuerpo
  return mensaje;                    // retorno
}

saludar('Ana');                      // invocación

Tipos de funciones según retorno: void y no void

Función void: se ejecuta por su efecto (mostrar, guardar, actualizar) y no devuelve un valor útil. Función no void: devuelve un valor que normalmente se usa en otra expresión o asignación.

Función void

function mostrarMensaje(nombre) {
  console.log(`Hola ${nombre}`);
  // no retorna valor útil
}

Función no void

function sumar(a, b) {
  return a + b; // retorna un number
}

const total = sumar(2, 3);

2Declaración y expresión

En JavaScript una función puede escribirse como declaración o como expresión asignada. Aquí aparece el hoisting (elevación): el motor registra ciertas declaraciones antes de ejecutar línea por línea.

Declaración

function saludar(nombre) {
  return `Hola ${nombre}`;
}

saludar('Ana');

Expresión / Arrow

const saludar = function(nombre) {
  return `Hola ${nombre}`;
};

const saludar2 = (nombre) => `Hola ${nombre}`;

Las funciones escritas como expresión pueden ser anónimas (sin nombre), por ejemplo al asignarlas a una constante o al pasarlas como argumento.

Una arrow con retorno implícito y una sola expresión es una forma de mínima expresión; en otros lenguajes se conoce como estilo lambda.

Funciones anónimas: dónde aparecen

setTimeout(function() {
  console.log('Han pasado 1000ms');
}, 1000);

const dobles = [1, 2, 3].map(function(n) {
  return n * 2;
});

const dobles2 = [1, 2, 3].map(n => n * 2); // arrow en mínima expresión (lambda)

Hoisting en variables y funciones

Caso Qué pasa antes de ejecutar Se puede usar antes de la línea
function foo() {} Se eleva nombre + cuerpo completo. Sí.
var x Se eleva solo la declaración. Queda en undefined. Sí, pero suele causar errores lógicos.
let x / const x Se elevan, pero quedan en zona temporal muerta (TDZ). No. Da ReferenceError.
const fn = () => {} La variable se eleva con TDZ; la función existe al asignar. No, hasta llegar a la asignación.

Declaración (sí se eleva)

saludar('Ana'); // funciona

function saludar(nombre) {
  return `Hola ${nombre}`;
}

Asignación con const (TDZ)

saludar('Ana'); // ReferenceError

const saludar = (nombre) => {
  return `Hola ${nombre}`;
};

¿Por qué hoy se prefiere asignar funciones a const?

  • Hace explícito el orden de ejecución: no dependes de llamar funciones "antes" por hoisting.
  • Evita reasignaciones accidentales del identificador de la función.
  • Mantiene un estilo uniforme con otras expresiones (callbacks, métodos cortos, funciones flecha).
  • Reduce errores de mantenimiento en archivos grandes al obligar a declarar antes de usar.

3Parámetros y valores por defecto

Puedes definir valores por defecto, capturar el resto de argumentos y usar destructuring.

Default + rest

function sumar(a = 0, b = 0) {
  return a + b;
}

function total(...nums) {
  return nums.reduce((acc, n) => acc + n, 0);
}

Destructuring

function crearUsuario({ nombre, edad }) {
  return { nombre, edad };
}

crearUsuario({ nombre: 'Ana', edad: 25 });

4Funciones callback

Una función callback es una función que se pasa como argumento a otra función para que esta la ejecute en un momento determinado.

Muchas callbacks se escriben como funciones anónimas (incluyendo funciones flecha), aunque también pueden ser funciones con nombre.

Ejemplo base

function ejecutarOperacion(a, b, callback) {
  return callback(a, b);
}

const resultado = ejecutarOperacion(10, 5, (x, y) => x - y);
// callback: (x, y) => x - y

Dónde se usan mucho

  • Métodos de arrays como map, filter y reduce.
  • Timers como setTimeout y setInterval.
  • Eventos del DOM (handlers) con addEventListener.
  • Operaciones asíncronas.

Ejemplos reales: timers y handlers

setTimeout(() => {
  console.log('Se ejecuta después de 1 segundo');
}, 1000); // callback

button.addEventListener('click', () => {
  console.log('Click detectado');
}); // callback (handler)

function onResize() {
  console.log('Ventana redimensionada');
}

window.addEventListener('resize', onResize);
window.removeEventListener('resize', onResize); // quitar handler

5Funciones HOF (Higher-Order Functions)

Una HOF es una función que recibe funciones como argumento, devuelve funciones, o ambas. Cuando recibe una función como argumento, esa función actúa como callback.

Importante: la HOF no es el callback; la HOF es la que orquesta y el callback es la lógica que se inyecta.

Sintaxis base

function procesar(datos, callback) {
  return callback(datos); // recibe función
}

function crearMultiplicador(factor) {
  return (n) => n * factor; // devuelve función
}

Uso clásico: reciben callback

const nums = [1, 2, 3, 4, 5];

const dobles = nums.map(n => n * 2);
const pares = nums.filter(n => n % 2 === 0);
const suma = nums.reduce((acc, n) => acc + n, 0);

Uso clásico: devuelven función

const por2 = crearMultiplicador(2);
const por10 = crearMultiplicador(10);

por2(7);  // 14
por10(7); // 70

Qué las hace especiales

  • Separan el qué del cómo: describes la transformación sin bucles manuales.
  • Mejoran reutilización: una HOF sirve para muchos casos cambiando solo el callback.
  • Facilitan composición: puedes encadenar operaciones pequeñas y legibles.
  • Encajan con closures para crear funciones configuradas (factory functions).

Cuándo usarlas

  • Procesamiento de arrays: map, filter, reduce.
  • Manejo de eventos y asincronía (callbacks, promesas, handlers y timers).
  • Crear utilidades reutilizables: validadores, formateadores, loggers, middlewares.

HOF en APIs del día a día

[1, 2, 3].map(n => n * 2); // map es HOF

setTimeout(() => console.log('tick'), 500); // recibe callback

form.addEventListener('submit', (e) => {
  e.preventDefault();
}); // addEventListener recibe callback

Errores comunes

  • Confundir HOF con callback: no son sinónimos. La HOF recibe/devuelve funciones; el callback es la función que se pasa.
  • Asumir que toda callback es HOF: una callback no "contiene" callbacks por defecto, solo es una función pasada como argumento.

6Scope y closures

El scope léxico determina qué variables están disponibles. Un closure es una función que recuerda su entorno.

Scope léxico

const nombre = 'Ana';

function saludar() {
  return `Hola ${nombre}`;
}

Closure

function contador() {
  let n = 0;
  return () => ++n;
}

const inc = contador();
inc(); // 1
inc(); // 2

7this y funciones flecha

Las funciones flecha no tienen su propio this; heredan el del contexto exterior.

Función normal

const obj = {
  nombre: 'Ana',
  saludar() {
    return this.nombre;
  }
};

Arrow

const obj = {
  nombre: 'Ana',
  saludar: () => this.nombre // this no es obj
};

Error común con this

No uses arrow como método de objeto cuando necesites el this del propio objeto.

8Buenas prácticas

  • Usa nombres verbales: getUser, createTask.
  • Evita funciones con demasiados parámetros; prefiere un objeto.
  • Divide lógica compleja en funciones pequeñas y puras.
