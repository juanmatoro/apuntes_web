1ESLint: reglas de calidad

ESLint detecta errores potenciales, malas prácticas y desviaciones de estilo antes de ejecutar la app.

npm install -D eslint
npx eslint .

2Prettier: formato consistente

Prettier no analiza lógica, pero elimina discusiones de estilo y mantiene formato homogéneo en todo el equipo.

npm install -D prettier
npx prettier . --write

3Comprobación de tipos

Aunque uses JavaScript, puedes activar chequeo con TypeScript para prevenir errores de interfaces y contratos.

// jsconfig/tsconfig con checkJs
{
  "compilerOptions": {
    "checkJs": true,
    "noEmit": true
  },
  "include": ["src/**/*"]
}

4Integración en CI

La clave es bloquear merges cuando el análisis falla: linter y type-check deben ser parte del pipeline.

{
  "scripts": {
    "lint": "eslint .",
    "format:check": "prettier . --check",
    "typecheck": "tsc --noEmit"
  }
}

5Herramientas complementarias

  • SonarQube/SonarCloud: métricas de deuda técnica, cobertura y code smells.
  • Husky + lint-staged: ejecutar checks antes de cada commit.
  • EditorConfig: homogeneizar saltos de línea e indentación entre editores.
