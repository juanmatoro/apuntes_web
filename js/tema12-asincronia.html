1Modelo asíncrono en JavaScript

JavaScript usa event loop: ejecución principal + colas de tareas para operaciones no bloqueantes.

console.log('A');
setTimeout(() => console.log('B'), 0);
console.log('C');
// salida: A, C, B

2Promesas

Una promesa representa el resultado futuro de una operación asíncrona.

fetch('/api/users')
  .then((res) => res.json())
  .then((data) => console.log(data))
  .catch((error) => console.error(error));

3async/await

Es sintaxis sobre promesas para escribir flujo asíncrono de forma más legible.

async function loadUsers() {
  const res = await fetch('/api/users');
  const users = await res.json();
  return users;
}

4Manejo de errores

En async/await, usa try/catch. En promesas usa .catch().

async function loadPosts() {
  try {
    const res = await fetch('/api/posts');
    if (!res.ok) throw new Error('HTTP error');
    return await res.json();
  } catch (error) {
    console.error('Fallo al cargar posts', error);
    return [];
  }
}

5Fetch y APIs JSON

La API fetch devuelve una promesa con la respuesta HTTP.

const payload = { nombre: 'Ana' };

await fetch('/api/users', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(payload)
});
