1Variables: let, const y var

Las variables son nombres simbólicos que apuntan (hacen referencia) a valores almacenados en memoria. En JavaScript se declaran con tres palabras clave:

Palabra clave Scope Reasignable Hoisting Redeclarable
let Bloque {} Sí Sí (TDZ) No
const Bloque {} No Sí (TDZ) No
var Función Sí Sí (undefined) Sí

TDZ (Temporal Dead Zone): con let y const, la variable existe en el scope desde el inicio del bloque, pero no es accesible hasta su declaración. Acceder antes lanza un ReferenceError. 

// let: variable reasignable, scope de bloque
let nombre = 'Ana';
nombre = 'Luis'; // OK

// const: NO reasignable, scope de bloque
const PI = 3.14159;
// PI = 3; // TypeError: Assignment to constant variable

// const con objetos: la referencia no cambia, pero el contenido sí
const user = { name: 'Ana' };
user.name = 'Luis'; // OK, se modifica el contenido
// user = {};       // TypeError

// var: scope de función, hoisting con valor undefined
console.log(x); // undefined (no lanza error)
var x = 10;

Buenas prácticas

  • Usar const por defecto
  • Usar let solo cuando sea necesario reasignar
  • Evitar var en código moderno
  • Nombres descriptivos en camelCase
  • Constantes globales en UPPER_SNAKE_CASE

2Tipos de datos

JavaScript es un lenguaje de tipado dinámico: las variables no tienen un tipo fijo, el tipo lo determina el valor asignado. Existen 7 tipos primitivos y el tipo Object.

Tipos primitivos

Tipo typeof Ejemplos
string "string" 'Hola', "mundo", `template`
number "number" 42, 3.14, NaN, Infinity
bigint "bigint" 9007199254740991n
boolean "boolean" true, false
undefined "undefined" undefined
null "object" * null
symbol "symbol" Symbol('id')

* typeof null devuelve "object" por un bug histórico de JS que se mantiene por compatibilidad.

Tipo Object

Todo lo que no es primitivo es un objeto: colecciones de pares clave-valor. Incluyen arrays, funciones, fechas, expresiones regulares, etc.

// Objeto literal
const persona = { nombre: 'Ana', edad: 25 };

// Array (objeto indexado)
const colores = ['rojo', 'verde', 'azul'];

// Función (objeto invocable)
const saludar = function(name) { return `Hola ${name}`; };

// typeof
typeof persona;  // "object"
typeof colores;  // "object"
typeof saludar;  // "function"

Primitivos vs Objetos

  • Primitivos: inmutables, se comparan por valor, se copian por valor
  • Objetos: mutables, se comparan por referencia, se copian por referencia
// Comparación por valor (primitivos)
'hola' === 'hola'  // true

// Comparación por referencia (objetos)
[1, 2] === [1, 2]  // false (distintas referencias)

const a = { x: 1 };
const b = a;       // misma referencia
a === b            // true

let x = 10;
x += 5;   // x = x + 5  → 15
x -= 3;   // x = x - 3  → 12
x *= 2;   // x = x * 2  → 24
x /= 4;   // x = x / 4  → 6
x %= 4;   // x = x % 4  → 2

Lógicos

Operador Descripción Comportamiento
&&AND lógicoDevuelve el primer valor falsy o el último
||OR lógicoDevuelve el primer valor truthy o el último
!NOT lógicoInvierte el valor booleano
??Nullish coalescingDevuelve el derecho si el izquierdo es null o undefined
 
// Short-circuit evaluation
const name = user && user.name;   // si user es falsy, devuelve user
const port = config || 3000;      // si config es falsy, devuelve 3000

// Nullish coalescing (??) vs OR (||)
0 || 'default'   // 'default' (0 es falsy)
0 ?? 'default'   // 0         (0 no es null ni undefined)

'' || 'default'  // 'default' ('' es falsy)
'' ?? 'default'  // ''        ('' no es null ni undefined)

// Optional chaining (?.)
const city = user?.address?.city; // undefined si alguno es null/undefined

Operador ternario

// condición ? valor_si_true : valor_si_false
const mensaje = edad >= 18 ? 'Adulto' : 'Menor';

4Coerción de tipos

La coerción es la conversión automática o explícita de un valor de un tipo a otro.

Coerción implícita (automática)

// String + number → concatenación
'5' + 3         // '53'
'5' + true      // '5true'

// Operadores numéricos → conversión a number
'5' - 3         // 2
'5' * 2         // 10
'5' / 1         // 5
true + 1        // 2
false + 1       // 1

// Contexto booleano
if ('hola') {}  // true (string no vacío)
if (0) {}       // false

Coerción explícita

// A String
String(42)        // '42'
(42).toString()   // '42'

// A Number
Number('42')      // 42
parseInt('42px')  // 42
parseFloat('3.14')// 3.14
+'42'             // 42 (operador unario +)

// A Boolean
Boolean(1)        // true
Boolean('')       // false
!!0               // false (doble negación)

Valores falsy y truthy

Valores falsy (7)

  • false
  • 0, -0
  • 0n (BigInt cero)
  • '' (string vacío)
  • null
  • undefined
  • NaN

Valores truthy

  • Todo lo que no es falsy
  • '0', 'false' (strings no vacíos)
  • [] (array vacío)
  • {} (objeto vacío)
  • Cualquier número distinto de 0
  • Cualquier función

5Modelo mental: variables y datos

Un modelo mental correcto de cómo funcionan las variables es clave para entender JavaScript. Las variables no “contienen” valores: son cables (wires) que apuntan a valores.

Variables como cables

  • Una variable es un nombre que apunta a un valor en el universo de JS
  • Los valores primitivos son únicos e inmutables: no se crean ni se destruyen
  • Los objetos sí se crean y pueden ser modificados (mutados)
  • Asignar una variable (=) cambia a dónde apunta el cable, no el valor en sí
// Primitivos: cada variable apunta al mismo valor único
let a = 'hola';
let b = 'hola';
a === b  // true (mismo valor en el universo JS)

// Objetos: cada literal crea un NUEVO objeto
let obj1 = { x: 1 };
let obj2 = { x: 1 };
obj1 === obj2  // false (distintos objetos)

Asignación vía variables

// Con primitivos: se copia el valor
let x = 10;
let y = x;    // y apunta a 10
x = 20;       // x apunta a 20, y sigue apuntando a 10
console.log(y); // 10

// Con objetos: se copia la REFERENCIA
let arr1 = [1, 2, 3];
let arr2 = arr1;        // arr2 apunta al MISMO array
arr1.push(4);
console.log(arr2);      // [1, 2, 3, 4]

Mutabilidad vs inmutabilidad

  • Inmutables (primitivos): no se pueden modificar. Las operaciones siempre devuelven un nuevo valor
  • Mutables (objetos): se pueden modificar directamente sus propiedades
// Strings son inmutables
let str = 'hola';
str.toUpperCase();    // devuelve 'HOLA' (nuevo valor)
console.log(str);     // 'hola' (no ha cambiado)

// Arrays son mutables
let arr = [1, 2, 3];
arr.push(4);          // modifica el array original
console.log(arr);     // [1, 2, 3, 4]
