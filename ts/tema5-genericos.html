Idea central Funciones genericas Restricciones

1Por que usar genericos

Un generico permite escribir logica reutilizable sin perder informacion de tipos.

2Funciones e interfaces genericas

function identidad<T>(valor: T): T {
  return valor;
}

const n = identidad(42);      // T = number
const s = identidad("hola");  // T = string

interface Caja<T> {
  valor: T;
}

const cajaUser: Caja<{ id: string }> = { valor: { id: "u1" } };

3Restricciones con extends

interface ConId {
  id: string;
}

function obtenerId<T extends ConId>(obj: T): string {
  return obj.id;
}

obtenerId({ id: "a1", nombre: "Marta" }); // OK
// obtenerId({ nombre: "Marta" }); // Error

Regla practica: usa nombres descriptivos para tipos genericos en codigo de dominio (TUsuario, TData).
