Interfaces Type aliases Comparativa

1Interfaces

Definen contratos para objetos, clases y funciones.

interface Usuario {
  id: string;
  nombre: string;
  activo?: boolean;
}

const u: Usuario = { id: "u1", nombre: "Ana" };

Las interfaces pueden extender otras interfaces:

interface Admin extends Usuario {
  permisos: string[];
}

2Type aliases

Los type sirven para nombrar cualquier tipo, no solo objetos.

type ID = string | number;
type Estado = "idle" | "loading" | "error";

type Producto = {
  sku: ID;
  nombre: string;
};

3Cuando usar cada uno

  • interface para contratos de objetos extensibles y APIs publicas.
  • type para uniones, intersecciones, tuplas y tipos derivados.
  • En la practica se combinan segun legibilidad y ergonomia.
type RespuestaOk = { ok: true; data: string[] };
type RespuestaErr = { ok: false; error: string };
type Respuesta = RespuestaOk | RespuestaErr;
