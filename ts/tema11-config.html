tsconfig Strict mode Paths y baseUrl

1tsconfig.json esencial

{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "Bundler",
    "rootDir": "./src",
    "outDir": "./dist",
    "strict": true,
    "sourceMap": true
  },
  "include": ["src"]
}

2Strict mode

  • noImplicitAny: evita parametros sin tipo.
  • strictNullChecks: obliga a tratar null y undefined.
  • noUncheckedIndexedAccess: mas seguridad en arrays y diccionarios.

3Aliases de rutas

{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@dominio/*": ["src/dominio/*"],
      "@ui/*": ["src/ui/*"]
    }
  }
}

Recuerda alinear aliases con el bundler (Vite/Webpack) y con el test runner.
