1Firma de una funcion

function sumar(a: number, b: number): number {
  return a + b;
}

const restar = (a: number, b: number): number => a - b;

Tipar retorno explicito es util en funciones publicas y librerias.

2Inferencia en funciones y contextual typing

TS infiere tipos de retorno y tambien tipos de parametros segun el contexto de uso.

// Inferencia de retorno (number)
const multiplicar = (a: number, b: number) => a * b;

// Contextual typing en callbacks
const nums = [1, 2, 3];
nums.map(n => n.toFixed(2)); // n es number por contexto

// Cuando no hay contexto, conviene anotar
const parse = (input: string): number => Number(input);

3Parametros opcionales, default y rest

function saludar(nombre: string, titulo?: string): string {
  return titulo ? `${titulo} ${nombre}` : nombre;
}

function crearId(prefijo: string = "USR"): string {
  return `${prefijo}-${Date.now()}`;
}

function media(...nums: number[]): number {
  return nums.reduce((a, b) => a + b, 0) / nums.length;
}

4Overloads

Permiten declarar varias firmas de llamada para una misma implementacion.

function formatear(valor: number): string;
function formatear(valor: Date): string;
function formatear(valor: number | Date): string {
  if (valor instanceof Date) return valor.toISOString();
  return valor.toFixed(2);
}

Evita overloads excesivos; si son demasiados casos, usa un objeto de opciones tipado.

5Ambito (scope), hoisting y paso de argumentos

  • Las declaraciones de funcion se elevan (hoisting); expresiones de funcion no.
  • El scope lexico define desde donde una variable es visible.
  • Primitivos se pasan por valor; objetos por referencia compartida.
saludo(); // OK
function saludo() { console.log("hola"); }

const persona = { nombre: "Ana" };
function renombrar(p: { nombre: string }) { p.nombre = "Luis"; }
renombrar(persona); // muta el objeto original

6Patrones de invocacion y this

Punto clave visto en el proyecto de clase: en JS/TS, this depende de como llamas a la funcion, no de donde esta escrita.

  • obj.metodo(): this es obj.
  • Llamada suelta fn(): en modo estricto, this es undefined.
  • Arrow function hereda this lexico: no sirve para metodos que dependan de contexto dinamico.
  • call/apply/bind fuerzan o fijan el contexto.
function presentar(this: { nombre: string }, ciudad: string): string {
  return `${this.nombre} - ${ciudad}`;
}

const user = { nombre: "Ana" };

presentar.call(user, "Madrid");      // "Ana - Madrid"
presentar.apply(user, ["Madrid"]);   // "Ana - Madrid"

const fija = presentar.bind(user);
fija("Sevilla");                     // "Ana - Sevilla"

Relacion con el proyecto: este patron aparece en ejemplos de herencia con funciones constructoras, donde se usa Person.call(this, ...) y Person.prototype.greet.apply(this).

7Des-estructuracion, spread/rest, callbacks y closures

Destructuring extrae valores de objetos/arrays en variables de forma declarativa.

const user = { id: "u1", nombre: "Ana", rol: "admin" };
const { nombre, ...resto } = user; // destructuring + rest

const nums = [10, 20, 30];
const [primero, , tercero] = nums; // destructuring de array

function printUser({ id, nombre }: { id: string; nombre: string }) {
  console.log(id, nombre); // destructuring en parametros
}

const copia = { ...user }; // spread (shallow copy)

function crearContador() {
  let total = 0;
  return () => ++total; // closure
}

const inc = crearContador();
inc(); // 1
inc(); // 2

Incluye tambien patrones de funciones anidadas, funciones de orden superior e IIFE para aislamiento puntual.

8structuredClone: clonacion profunda real

structuredClone() crea una copia profunda de estructuras complejas (objetos anidados, arrays, Map, Set, etc.).

const original = {
  user: { nombre: "Ana" },
  tags: ["ts", "js"]
};

const deepCopy = structuredClone(original);
deepCopy.user.nombre = "Luis";

console.log(original.user.nombre); // "Ana" (no se modifica)
Tecnica Tipo de copia Uso recomendado
{ ...obj } / [...arr]ShallowCopias simples sin anidamiento profundo.
JSON.parse(JSON.stringify(x))Deep parcialSolo datos serializables JSON (pierde tipos especiales).

Limite importante: no clona funciones ni nodos DOM; para esos casos usa patrones especificos de serializacion o reconstruccion.
