1Firma de una funcion

function sumar(a: number, b: number): number {
  return a + b;
}

const restar = (a: number, b: number): number => a - b;

Tipar retorno explicito es util en funciones publicas y librerias.

2Inferencia en funciones y contextual typing

TS infiere tipos de retorno y tambien tipos de parametros segun el contexto de uso.

// Inferencia de retorno (number)
const multiplicar = (a: number, b: number) => a * b;

// Contextual typing en callbacks
const nums = [1, 2, 3];
nums.map(n => n.toFixed(2)); // n es number por contexto

// Cuando no hay contexto, conviene anotar
const parse = (input: string): number => Number(input);

3Parametros opcionales, default y rest

function saludar(nombre: string, titulo?: string): string {
  return titulo ? `${titulo} ${nombre}` : nombre;
}

function crearId(prefijo: string = "USR"): string {
  return `${prefijo}-${Date.now()}`;
}

function media(...nums: number[]): number {
  return nums.reduce((a, b) => a + b, 0) / nums.length;
}

4Overloads

Permiten declarar varias firmas de llamada para una misma implementacion.

function formatear(valor: number): string;
function formatear(valor: Date): string;
function formatear(valor: number | Date): string {
  if (valor instanceof Date) return valor.toISOString();
  return valor.toFixed(2);
}

Evita overloads excesivos; si son demasiados casos, usa un objeto de opciones tipado.
