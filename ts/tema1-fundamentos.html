1Que es TypeScript

TypeScript es un superset de JavaScript que anade un sistema de tipos estatico y herramientas de analisis en tiempo de desarrollo.

  • Se escribe codigo .ts y se compila a .js.
  • Ayuda a detectar errores antes de ejecutar la aplicacion.
  • Mejora el autocompletado y la refactorizacion en el editor.
function saludar(nombre: string): string {
  return `Hola ${nombre}`;
}

saludar("Ana"); // OK
// saludar(42); // Error de tipos

2Flujo de trabajo

  1. Escribes codigo TypeScript.
  2. El compilador tsc valida tipos.
  3. Se genera JavaScript compatible con tu entorno objetivo.
  4. Ejecutas el resultado con navegador o Node.js.
npx tsc --init
npx tsc
node dist/index.js

4Buenas practicas iniciales

  • Activar strict desde el primer dia.
  • Evitar any salvo casos justificados.
  • Separar codigo fuente (src) de salida (dist).
  • Usar ESLint + Prettier para mantener consistencia.

Lenguajes en la web y evolucion del estandar ECMAScript

  • HTML estructura contenido, CSS lo presenta y JS/TS agrega logica y comportamiento.
  • TypeScript se apoya en ECMAScript y compila a JavaScript estandar.
  • Versiones clave: ES5 (base legacy), ES6 (modulos, clases, let/const), ES2017 (async/await), ES2020+ (mejoras continuas).

Estructura del lenguaje

  • Sintaxis: sentencias, expresiones, bloques y palabras reservadas.
  • Identificadores claros y consistentes (camelCase, PascalCase para tipos/clases).
  • Modo estricto: evita errores silenciosos y mejora la seguridad del codigo.

Entornos JS y herramientas

  • Navegador: consola, DOM y Web APIs.
  • Node.js: scripts, backend, tooling de build/test.
  • Playgrounds: utiles para experimentar rapido con tipos y snippets.

Modelo mental de datos

  • Primitivos vs referenciados, mutabilidad y paso por valor/referencia.
  • Variables con let/const, evitando magic numbers.
  • Casting/coercion: conversiones explicitas preferibles a conversiones implicitas.

6Referencias recomendadas

  • Documentacion oficial de TypeScript (handbook y tsconfig).
  • MDN para JavaScript, Web APIs y asincronia.
  • ECMAScript spec para semantica formal del lenguaje.
  • Recursos de fundamentos y programacion funcional/asincrona para reforzar base de JS.
