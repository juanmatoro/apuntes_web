1Tipos primitivos

Los mas usados son string, number y boolean.

let nombre: string = "Marta";
let edad: number = 30;
let activo: boolean = true;

Tambien existen null, undefined, symbol y bigint.

2Arrays y tuplas

// Array de strings
const tags: string[] = ["ts", "js", "web"];

// Tupla: orden y tipo fijo por posicion
const punto: [number, number] = [10, 20];
  • Usa arrays cuando la longitud es variable.
  • Usa tuplas cuando cada posicion tiene significado concreto.

3Inferencia de tipos

TypeScript puede inferir tipos sin anotarlos manualmente cuando el contexto es claro.

let ciudad = "Madrid";   // string
let anio = 2026;         // number
let publicado = true;    // boolean

const notas = [8, 9, 10]; // number[]

// Si no hay contexto, conviene anotar:
let valor; // any (si noImplicitAny esta desactivado)
  • La inferencia reduce ruido, pero no sustituye contratos en APIs publicas.
  • Anota tipos cuando el dominio no es obvio o hay ambiguedad.

4Scope de bloque y variables repetidas

Un bloque { ... } crea un nuevo scope para let y const, permitiendo reutilizar nombres sin colisionar.

const estado = "global";

{
  const estado = "local"; // variable distinta (shadowing)
  console.log(estado); // "local"
}

console.log(estado); // "global"

Evita abusar del shadowing si reduce legibilidad, pero es util para delimitar contexto.

5Tipos especiales: any, unknown, void, never

Tipo Uso recomendado
anyEvitarlo; desactiva seguridad de tipos
unknownDatos externos que deben validarse antes de usar
voidFunciones sin retorno util
neverFunciones que nunca terminan o siempre lanzan error
 
function fail(msg: string): never {
  throw new Error(msg);
}

function log(msg: string): void {
  console.log(msg);
}
