Condicionales infer Branded types Asincronia

1Tipos condicionales

type EsString<T> = T extends string ? true : false;

type A = EsString<"hola">;  // true
type B = EsString<42>;      // false

Permiten modelar reglas de tipos segun condiciones declarativas.

2Extraccion con infer

type TipoRetorno<T> = T extends (...args: any[]) => infer R ? R : never;

type Fn = (x: number) => Promise<string>;
type R = TipoRetorno<Fn>; // Promise<string>

3Branded types para dominio

type Email = string & { readonly __brand: "Email" };

function crearEmail(valor: string): Email {
  if (!valor.includes("@")) throw new Error("Email invalido");
  return valor as Email;
}

Este patron evita mezclar strings comunes con valores de dominio criticos.

4Asincronia en JS/TS: repaso completo

Event loop, concurrencia y callbacks

  • JS ejecuta en un hilo principal con modelo concurrente via event loop.
  • Callbacks asincronas: timers, eventos, I/O.
  • Problema clasico: callback hell por anidacion profunda.

Promesas y microtasks

const p = new Promise<number>((resolve) => setTimeout(() => resolve(42), 100));
p.then(v => console.log(v)).catch(err => console.error(err));
  • Encadenamiento con then/catch/finally.
  • Metodos estaticos: Promise.all, allSettled, race, any.

Async/await, top-level await y fetch

async function getUsers(): Promise<unknown> {
  const resp = await fetch("https://jsonplaceholder.typicode.com/users");
  if (!resp.ok) throw new Error("HTTP error");
  return resp.json();
}

const data = await getUsers(); // top-level await (si el entorno lo permite)

Top level await permite usar await en el modulo raiz cuando el entorno lo soporta.

Promisificacion: convertir APIs de callback a promesas para simplificar flujo y manejo de errores.

En consumo de Rest APIs, combina tipado de respuesta, validacion y manejo de errores HTTP.
