Ramas Commits Pull Request

1Estrategia de ramas

Modelo simple recomendado: `main` estable, `dev` integracion y `feature/*` para cada cambio concreto.

2Commits utiles

git checkout -b feature/login-form
git add .
git commit -m "feat(front): valida email y password en login"

Un commit debe representar una unidad de cambio comprensible.

3Revision y merge

En PR revisa impacto funcional, riesgos, deuda tecnica y pruebas. No hagas merge sin CI en verde.
