Vision Capas Flujo request

1Que significa Fullstack

Fullstack no es saber todo a nivel experto; es poder conectar front, back y BD con un criterio consistente.

2Arquitectura por capas

UI (Front)
  -> API HTTP (Back)
    -> Servicio / Dominio
      -> Repositorio
        -> Base de Datos

Separar capas facilita pruebas, mantenimiento y evolucion del proyecto.

3Flujo de una peticion

Cliente envia request, backend valida y aplica reglas de negocio, repositorio consulta BD y se devuelve una respuesta tipada al front.
