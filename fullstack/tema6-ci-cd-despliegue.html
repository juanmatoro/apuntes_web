CI CD Observabilidad

1Integracion continua

Cada push ejecuta lint, tests y build. Si algo falla, el cambio no debe integrarse.

2Despliegue continuo

Promociona artefactos entre entornos (dev, staging, prod) con controles y rollback definido.

build -> test -> package -> deploy staging -> smoke tests -> deploy prod

3Observabilidad minima

  • Logs estructurados.
  • Metricas de latencia, errores y uso.
  • Alertas sobre SLO basicos.
