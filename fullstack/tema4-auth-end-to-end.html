Login Tokens Roles

1Flujo de login

El frontend envia credenciales por HTTPS. El backend valida y emite token si son correctas.

2Access y refresh token

  • Access token corto para llamadas API.
  • Refresh token para renovar sesion de forma controlada.

3Autorizacion por rol

if (!user.roles.includes("admin")) {
  return res.status(403).json({ error: "FORBIDDEN" });
